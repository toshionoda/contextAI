### 2. 9.4.1 差分ハイライトの見方【書き換え】

**現状**：赤・青・グレーの説明のみ

**実態**：カラー凡例がUIに常時表示、「変更箇所のみ表示」モードが追加

```markdown
#### 9.4.1 差分ハイライトの見方

差分は色で区別されます。ファイル名ヘッダーにもカラー凡例が常時表示されているため、
画面上でいつでも確認できます。

- **赤色**：新版で追加された箇所（図形、テキスト、注釈など）
- **青色**：旧版に存在したが新版では削除された箇所
- **グレー**：両方のファイルに同じ内容で存在する部分（変更なし）

#### 9.4.2 「変更箇所のみ表示」モード

- **OFF（デフォルト）**：グレーのマスキングで全体の差分を俯瞰
- **ON**：変更箇所だけに集中して確認

変更が多い図面で、具体的にどこが変わったかを素早く把握したい場合に便利です。
```

- html2canvasはoklch()カラー関数に対応していないため使用しない  → スクリーンショットベースのキャプチャを使用- MediaRecorderはMP4非対応 → VideoEncoder + mp4-muxerを使用- file inputの重複イベントリスナー → 毎回要素を再作成（window._createFI）- 複数のfile inputがある場合 → find ツールで都度 unnamed の ref を取得- ボタンラベルが画面端で切れる場合 → requestAnimationFrameで位置チェック、right/leftで調整- ダイアログ閉→開でReactが全DOM再レンダリング → 名前置換の再実行が必須- CiviLinkホームページ（[civilink.ai](http://civilink.ai/)）はアクセス不可の場合がある  → アプリのブルー系カラー（#3B82F6）をデザイン参考にする- 字幕が画面幅を超える場合 → テキストを短縮して1行に収める- encoder/muxer/target は必ず window に保存する（スコープ消失防止）
