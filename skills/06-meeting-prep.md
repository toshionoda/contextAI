# 06 - 顧客MTG準備

## 目的
顧客MTG前にコンテキストを収集し、効果的なアジェンダを作成する。

## 実行タイミング
- 顧客MTGの前日〜当日

## 手順

### 1. 顧客コンテキスト確認
- `customers/{顧客名}/README.md` で全体像を把握
- 直近の課題・要望・意思決定ログを確認
- ヘルススコアの確認

### 2. 直近のやり取り確認
- Slackでの直近のやり取り
- 前回MTGの議事録（`customers/{顧客名}/meetings.md`）
- 前回MTGからのアクション進捗

### 3. プロダクト情報の確認
- 直近のリリースノート（`product/release-notes/`）
- 顧客に関連する既知の課題の対応状況
- ロードマップの更新

### 4. アジェンダ作成
```
## MTGアジェンダ: {{顧客名}} - YYYY/MM/DD
1. 前回アクションの確認
2. 利用状況の確認
3. 課題・要望のディスカッション
4. プロダクトアップデート共有
5. 次回アクションの確認
```

### 5. 準備物の確認
- デモ環境の準備（必要な場合）
- 資料・スクリーンショットの準備
- 技術的な回答の準備

## 出力
- MTGアジェンダ
- 顧客コンテキストサマリー
- 共有予定のプロダクトアップデート
