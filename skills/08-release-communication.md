# 08 - リリース案内作成

## 目的
新機能・修正のリリース時に、顧客向けの案内を作成する。

## 実行タイミング
- プロダクトリリース後

## 手順

### 1. リリース内容の確認
- `product/release-notes/` から最新リリース情報を取得
- 変更点の技術的影響を評価

### 2. 顧客別の影響評価
- 各顧客の利用状況に照らして影響を評価
- 特に恩恵を受ける顧客を特定
- 注意が必要な変更（破壊的変更等）を特定

### 3. 案内文作成
- 全体向けリリースノート
- 顧客別のカスタマイズメッセージ（必要な場合）

### 4. 配信
- Slackでの共有
- 個別フォローが必要な顧客への連絡

## 案内テンプレート
```
## CiviLink アップデートのお知らせ（YYYY/MM/DD）

### 新機能
-

### 改善
-

### バグ修正
-

### お客様へのご案内
-

ご不明点がございましたらお気軽にお問い合わせください。
```

## 注意点
- 技術的な変更は顧客にわかりやすい言葉で説明
- DocuWorks関連の変更は特に丁寧に説明（期待値管理）
