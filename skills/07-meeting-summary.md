# 07 - MTGサマリー・タスク抽出

## 目的
MTG後に議事録を要約し、タスクを抽出・追跡する。

## 実行タイミング
- 顧客MTG終了後（当日中が望ましい）

## 手順

### 1. 議事録の整理
- MTGの要点を構造化
- 参加者・日時の記録

### 2. タスク抽出
- 自社側のアクションアイテム
- 顧客側のアクションアイテム
- 期限の設定

### 3. 顧客情報の更新
- `customers/{顧客名}/meetings.md` にMTG記録を追加
- `customers/{顧客名}/README.md` の意思決定ログを更新
- 必要に応じてヘルススコアを更新

### 4. タスクの登録
- `tasks/todo.md` にアクションアイテムを追加
- 緊急度に応じてセクション分け

### 5. 機能要望の整理（該当する場合）
- MTGで出た機能要望は `04-feature-request.md` のフローで整理

## 出力テンプレート
```
## MTGサマリー: {{顧客名}} - YYYY/MM/DD
参加者:
所要時間:

### 要点
1.
2.

### 決定事項
-

### アクションアイテム
| タスク | 担当 | 期限 |
|--------|------|------|
|        |      |      |

### 次回MTG
- 予定日:
- 議題候補:
```
