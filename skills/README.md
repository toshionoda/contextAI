# skills/ - 業務スキル定義

CiviLink CS × Civil Engineerとしての繰り返し業務を型化。

## スキル一覧

| # | スキル | 用途 | 頻度 |
|---|--------|------|------|
| 00 | 始業チェック | 朝の状況把握・優先度整理 | 毎日 |
| 01 | 顧客問い合わせ対応 | 技術的・機能的な問い合わせ処理 | 日常 |
| 02 | オンボーディング支援 | 新規顧客の導入ガイド | 随時 |
| 03 | 顧客ヘルスチェック | 利用状況の定期評価 | 週次 |
| 04 | 機能要望整理 | 顧客フィードバックの構造化 | 随時 |
| 05 | 技術調査 | 技術課題の調査・解決策検討 | 随時 |
| 06 | 顧客MTG準備 | MTG前のコンテキスト収集・アジェンダ作成 | MTG前 |
| 07 | MTGサマリー | 議事録要約・タスク抽出 | MTG後 |
| 08 | リリース案内作成 | 新機能・修正のリリース通知 | リリース時 |

## スキルチェーン（連続実行）

### 顧客MTGフロー
```
06-meeting-prep → [MTG実施] → 07-meeting-summary → 04-feature-request（必要に応じて）
```

### 課題解決フロー
```
01-customer-inquiry → 05-technical-investigation → 04-feature-request（機能不足の場合）
```
