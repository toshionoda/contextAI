# 04 - 機能要望整理

## 目的
顧客からの機能要望を構造化し、プロダクトチームへフィードバックする。

## 実行タイミング
- 顧客から機能要望を受けた時
- MTGで要望が出た時

## 手順

### 1. 要望の記録
- 顧客名、担当者、日付
- 要望内容（顧客の言葉そのまま）
- 背景・理由（なぜその機能が必要か）

### 2. 分類
- カテゴリ: UI/UX / 図面表示 / 照査機能 / 連携 / パフォーマンス / その他
- 優先度: P0（ブロッカー）/ P1（重要）/ P2（あると嬉しい）/ P3（将来的に）

### 3. 技術的実現可能性の評価
- 既存機能で対応可能か
- 技術的制約はあるか（例: DocuWorks SDK制約）
- 代替アプローチの提案

### 4. 影響範囲の評価
- 同じ要望を持つ他の顧客
- 有償化への影響
- ロードマップとの整合性

### 5. プロダクトチームへのフィードバック
- `product/features/` にフィーチャーリクエストとして記録
- Slack等でプロダクトチームに共有

## 出力テンプレート
```
## 機能要望: {{タイトル}}
- 顧客: {{顧客名}}
- 日付: YYYY/MM/DD
- カテゴリ:
- 優先度:
- 要望内容:
- 背景・理由:
- 技術的評価:
- 影響範囲:
- 推奨アクション:
```
