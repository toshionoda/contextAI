# 図面照査ワークフロー

## 一般的な土木設計の図面照査フロー

### 従来フロー
1. 設計者がCADで図面作成
2. DocuWorksまたはPDFに変換
3. 照査者が印刷 or 画面上で確認
4. 朱書き（赤入れ）で指摘
5. 設計者が修正
6. 再照査

### CiviLink導入後のフロー
1. 設計者がCADで図面作成
2. PDF（推奨）をCiviLinkにアップロード
3. 照査者がCiviLink上でコメント・マークアップ
4. 設計者がCiviLink上で指摘を確認・修正
5. 修正版をアップロード・再照査

## 照査部門への導入のポイント
- 数百枚の図面をストレスなく閲覧できる体験が必須
- ベクター表示が照査品質に直結（拡大時の鮮明さ）
- 既存のDocuWorksワークフローからの移行コストを最小化

## 課題
- DocuWorks運用からの移行抵抗
- 大量図面のアップロード・管理の効率化
