# 公共工事・建設業における書類自動チェック・基準適合確認 調査レポート

**調査日:** 2026-04-09
**目的:** 自治体向けCiviLink提案に向けた、書類自動チェック・基準適合確認の技術・サービスの現状把握
**結論:** 電子納品の形式チェックは成熟済み、設計図書の内容チェック（積算・照査）はAI活用が急速に進展中、自治体職員の負担軽減に直結する領域は「積算ミス検知」と「設計図書の整合性チェック」

## 1. 電子納品チェックシステムの現状

### 1-1. 国交省の電子納品チェックシステム

**最新バージョン（2026年4月改定）:**
- 土木: Ver.20.0.0
- 電通: Ver.14.0.0
- 機械: Ver.12.0.0

**チェック内容:**

| チェック項目 | 内容 |
|---|---|
| フォルダ構成 | 電子納品要領に定められた標準フォルダ構成（REPORT, DRAWING, PHOTO等）に従っているか |
| ファイル命名規則 | 半角英数大文字、ファイル名8文字以内、拡張子3文字以内等 |
| XMLメタデータ | 管理ファイル（INDEX_D.XML / INDEX_C.XML）の必須要素（工事情報・成果区分・媒体情報等）の記載 |
| ファイル形式 | CADデータはSXF(P21/SFC)形式か、写真はJPEG形式か等 |
| 整合性 | XMLに記載のないフォルダ・ファイルの有無 |
| i-Constructionデータ | 「ICON」フォルダの有無（最新版で追加） |

**特徴:**
- 無償で提供（国交省NILIM開発）
- あくまで**形式チェック**であり、書類の**内容の正しさ**はチェックしない
- 2025年3月にオンライン電子納品実施要領（工事編・業務編）を公開 → CD-R納品からの移行を推進
- ダウンロード先: https://www.cals-ed.go.jp/

### 1-2. 自治体が利用しているチェックシステム

| 自治体/機関 | システム | 備考 |
|---|---|---|
| 国交省・地方整備局 | 電子納品チェックシステム（国交省版） | 上記の標準システム |
| 農林水産省 | 電子納品チェックシステム（農水版） | 農業農村整備事業に特化 |
| 東京都 | 独自の電子納品要領に基づくチェック | 都の要領に準拠 |
| 各県 | 多くは国交省版を基本とし、県独自の追加要件を付加 | 山梨県・山形県等が独自ガイドライン公開 |

**自治体職員の負担:**
- 電子納品の形式チェック自体は自動化されている
- **問題は「チェックを通っても内容が正しいかは別」** → 内容の確認は人力
- フォーマットがバラバラで横断的な活用に至っていない（北九州市ヒアリングでも確認済み）

## 2. 施工計画書・出来形書類の自動チェック事例

### 2-1. 施工計画書のAI自動作成・チェック

#### EARTHBRAIN「工事書類AI」（2026年4月提供開始）
- コマツ系列のEARTHBRAINが提供
- 生成AIで施工計画書のドラフトを自動生成
- 従来5日以上かかっていた作成業務を大幅短縮
- 施工計画書データを会社の資産として一元管理
- ベテラン依存を解消、若手でも高品質な書類作成が可能

#### NITACO「施工計画書AI」（2026年2月ベータ版公開）
- 建設業法で定められた**全29業種に対応**
- 従来2〜3日→**最短30分**で初稿作成（最大90%工数削減）
- 発注者の施工管理基準、数量計算書等をアップロード → AIが内容解析してドラフト生成
- 出来形・品質・写真管理基準一覧表も自動作成

#### 大成建設「全体施工計画書作成支援システム」（2025年11月発表）
- 視覚言語モデル（VLM）ベースのマルチモーダル生成AIを活用
- 土木工事における全体施工計画書案を**約10分で自動作成**
- 社内実務で検証済み

**現状の限界:**
- 「作成支援」が主流で、「既存書類の自動チェック（基準適合確認）」まで踏み込んだ製品はまだ少ない
- 共通仕様書との自動照合機能は開発途上

### 2-2. 出来形管理の自動チェック

| 技術 | 内容 | 実用化状況 |
|---|---|---|
| ドローン測量 + AI解析 | 撮影データから出来形を自動マッピング、計画との差異を検出 | 実用化進展中 |
| 点群データ vs 設計図面 | LiDAR点群と設計データを自動比較、許容偏差を超える箇所を検出 | Avvirが±3mm精度を達成 |
| スマホ配筋検査 | 写真から鉄筋本数・間隔をAI自動確認 | 清水建設で実用化（5分→20〜30秒） |
| カメラ画像解析 | i-Construction 2.0が推進する施工管理のオートメーション化 | 2027年度以降の3Dモデル契約図書化に向けて拡大 |

### 2-3. 品質管理書類の自動チェック

- 現時点では「自動作成」の段階。**内容の正しさを自動検証**するシステムは限定的
- 建設技術研究所が図面ファイルと計算結果の整合性を自動チェックするシステムを開発（後述）

## 3. 図面の基準チェック自動化

### 3-1. CAD製図基準への自動チェック

| システム | 提供元 | チェック内容 |
|---|---|---|
| Autodesk SXF Viewer | オートデスク | 線の色・種類・太さのCAD製図基準準拠確認、重複図形・ショートベクトル確認 |
| EX-TREND武蔵 建設CAD | 福井コンピュータ | CAD製図基準（案）準拠チェック、地域ルール対応、エラー箇所の赤入れ自動表示 |
| SXFブラウザ | 国交省（開発終了） | CADデータの表示・確認用。チェック機能は限定的。市販ソフトに移行 |

**現状:**
- CAD製図基準の**形式的なチェック**（レイヤー名、線種、文字サイズ等）は自動化されている
- **内容の妥当性**（寸法の正しさ、注記の整合性等）のチェックは自動化が困難で、目視に依存

### 3-2. 設計図書の整合性チェック（内容レベル）

#### 東芝デジタルソリューションズ「CISSART」
- **最も注目すべきシステム**
- AIで設計図面から工種名称・規格・数量を自動抽出
- 数量総括表との対比・整合性チェックを自動実行
- エラーリストを出力、図面上に赤/黄チェックマークを表示
- 北陸地方整備局のi-Constructionで採用実績あり
- 対象: 道路・河川の維持修繕工事の設計図書
- URL: https://www.global.toshiba/jp/products-solutions/public-ict/icon.html

#### 建設技術研究所 自動チェックシステム（2025年4月公開）
- 図面ファイルと計算結果の文書ファイルを読み込み、整合性を自動チェック
- 「照査表」と「赤黄チェック」を自動出力
- 従来2〜3日かかる照査が**数十分で完了**
- 社内システムとして開発、今後**社外へのサービス提供も検討**
- URL: https://www.ctie.co.jp/news/tech/2025/20250414_1483.html

#### 八千代エンジニヤリング 設計照査ソフトウェア（2022年オープンソース公開）
- 国交省の詳細設計照査要領に準拠
- BIM/CIMモデル（LandXML形式）から情報を抽出し、設計計算書との整合性を自動確認
- 道路設計に対応
- **オープンソースとして公開** → 他社が活用・改良可能
- URL: https://www.yachiyo-eng.co.jp/news/2022/06/post_645.html

### 3-3. 積算チェックのAI自動化

#### 富士通「ESTIMA」AI積算チェック
- AI「Zinrai」を活用、過去の積算データから類似工事を抽出・比較
- 積算の誤りの可能性がある箇所を**自動検知・通知**
- **長野県の実証実験**: 2,105件の道路改良工事 → 87件を要チェックと検知 → うち24件で実際に誤りを発見
- 自治体向け積算システムとして導入実績あり

### 3-4. 建築分野の図面基準チェック（参考）

#### mign「revisio」および建築確認AI
- CAD/BIM図面の修正箇所をAIで自動検出（2025年7月リリース）
- 建築確認申請ルールの自動判定システム（横須賀市と実証実験）
- 3D建物形状から斜線制限・日影規制・建蔽率/容積率を自動計算・判定
- 2026年4月からBIMベースの建築確認審査が開始予定
- 2.3億円を調達（2025年）、不動産・建設特化型AIエージェント
- URL: https://www.mign.io/

#### 国交省「建築確認申請図書作成支援サービス」（2025年11月公開）
- 建築基準法改正（2025年4月）への対応として無償公開
- AIが建築確認申請図書の作成をサポート

### 4-3. RAG（検索拡張生成）を使った基準適合チェック

**現時点で実用化されているRAGベースの基準チェックシステムは確認できず。** ただし、以下の関連動向がある：

- **Civils.ai**（海外）: 構造設計の自動化エージェント。設計基準照合・荷重計算・配筋設計をLLMで実行
- **CiviLinkの構想**: 国交省設計基準・道路構造令をRAGで参照し、指摘箇所の基準適合性を回答するLLM照査アシスタント（プロダクト計画段階）
- **デジタル庁**: 政府AI活用戦略「ガバメントAI」推進中。自治体業務へのLLM導入は急速に進展（指定都市90%・都道府県87%が生成AI導入済み）
- **富士通「Takane」**: 中央省庁でパブリックコメント業務効率化の実証実験実施（2025年）

## 5. CiviLinkと書類チェック自動化の統合可能性

### 5-1. 既存プラットフォームに書類チェック機能を追加した事例

| プラットフォーム | チェック機能追加 | 方法 |
|---|---|---|
| Photoruction | 電子納品機能（福井コンピュータ連携） | データ連携 |
| Autodesk Construction Cloud | Construction IQ（品質リスク予測） | 1.5億件のデータでAI学習 |
| Procore | Helix（旧Copilot）でRFI・仕様書をAI解析 | LLM統合 |
| 福井コンピュータ EX-TREND | CAD製図基準チェック内蔵 | ネイティブ機能 |

### 5-2. CiviLinkで実現可能性が高い書類チェック機能

**即時実現可能（CiviLinkの既存資産活用）:**

| 機能 | 概要 | 自治体メリット |
|---|---|---|
| **差分比較→指摘自動生成** | 図面バージョン間の差分を検出し、変更箇所ごとにドラフト指摘を自動生成 | チェック漏れ防止、照査時間短縮 |
| **バージョン影響伝播** | 図面更新時に既存指摘への影響を自動検出、再照査フラグを付与 | 改訂時の確認漏れ防止 |
| **電子納品フォルダ構成チェック** | CiviLink上の図面がそのまま電子納品要領準拠のフォルダ構成になっているか確認 | 納品前の手戻り防止 |

**中期的に実現可能（AI開発が必要）:**

| 機能 | 概要 | 自治体メリット |
|---|---|---|
| **図面⇔数量計算書の整合性チェック** | AI+OCRで図面から数量を抽出し、数量計算書と自動対比 | 積算ミスの早期発見（北九州市の最大ニーズ） |
| **LLM照査アシスタント** | 設計基準をRAGで参照し、図面上の指摘箇所の基準適合性を回答 | 技術者不足の補完 |
| **予測品質ヒートマップ** | 過去の照査データから品質リスクゾーンを図面上に可視化 | 重点チェック箇所の効率的特定 |

**長期的に実現可能（データ蓄積が前提）:**

| 機能 | 概要 | 自治体メリット |
|---|---|---|
| **積算ミス検知AI** | 過去の積算データを学習し、異常値を自動検知（富士通ESTIMAモデル） | 桁間違い等の重大ミス防止 |
| **点群⇔設計図面の自動比較** | 出来形管理基準値に基づく許容偏差チェック | 出来形検査の省力化 |
| **照査報告書自動生成** | 累積指摘データから国交省様式の照査報告書を自動生成 | 書類作成時間の大幅削減 |

## 6. 自治体職員が楽になる施策（実現可能性順）

### 優先度1: すぐに効果が出る（半年以内）

1. **電子納品前の形式チェック支援**
   - CiviLinkにアップロードした図面が電子納品要領の命名規則・フォルダ構成に準拠しているか自動確認
   - 「納品直前の手戻り」を防止
   - 実現難易度: 低（ルールベースで実装可能）

2. **図面バージョン差分の自動可視化**
   - 設計変更があった箇所を自動ハイライト
   - 「何が変わったか」を職員が即座に把握
   - CiviLinkの既存差分比較エンジンを活用
   - 実現難易度: 低（既存機能の拡張）

### 優先度2: 中期的に実現（半年〜1年）

3. **図面と数量計算書の整合性チェック**
   - AI+OCRで図面から工種・規格・数量を読み取り、数量計算書と自動対比
   - **北九州市が最も関心を示している領域**
   - 小野田ケミコでの実績（図面→数量→単価の自動チェック）を自治体向けに展開
   - 実現難易度: 中（OCR精度とドメイン知識が鍵）

4. **CAD製図基準の自動チェック結果の統合表示**
   - 既存のCAD製図基準チェックツール（福井コンピュータ等）の結果をCiviLink上で一元表示
   - 「基準チェック結果→指摘→是正→完了」のワークフローを一気通貫に
   - 実現難易度: 中（データ連携の設計が必要）

### 優先度3: 長期的に実現（1年以上）

5. **RAGベースの基準適合アシスタント**
   - 共通仕様書・設計基準をベクトルDBに格納
   - 「この排水勾配は基準を満たしているか？」に根拠条文付きで回答
   - 技術者不足の補完として自治体に極めて有効
   - 実現難易度: 高（基準書のデータ化と精度検証が必要）

6. **積算ミス検知AI**
   - 過去の積算データの蓄積が前提
   - 富士通ESTIMA + 長野県の成功事例を参考に、CiviLink上で積算チェック結果を可視化
   - 実現難易度: 高（自治体の積算システムとの連携が必要）

```
    SXF Viewer          |   富士通ESTIMA       |
    福井コンピュータ    |   mign（建築分野）   |
                        |   CiviLink照査AI     |
                        |                    |
    ルールベース ←――――→ AI/LLMベース
```

### CiviLinkの差別化ポイント

1. **土木設計図面に特化した唯一のクラウドプラットフォーム**
   - 建築分野のmign、施工段階のANDPAD/Photoructionとは棲み分け
2. **差分比較エンジン + バージョン管理が既存資産**
   - 差分→指摘自動生成は業界初の可能性
3. **照査ワークフロー内蔵**
   - チェック結果を指摘として管理→是正→完了の一気通貫
4. **土木ドメイン知識（Malmeの20名の土木技術者）**
   - 基準書のRAG構築に不可欠なドメイン理解
