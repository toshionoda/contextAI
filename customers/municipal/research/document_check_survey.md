# 自治体における公共工事・業務委託の書類チェック調査

**調査日:** 2026-04-09
**目的:** 自治体が共通仕様書・基準に基づいてチェック・確認する書類の全体像を把握し、CiviLinkの自治体向け提案に活用する

---

## 1. 公共工事で自治体が確認する書類の一覧

### 1-1. 施工計画書

**根拠:** 土木工事共通仕様書 第1編 第1章 第1節（各地整・自治体版で条番号は異なる）

施工計画書は工事着手前に受注者が監督職員に提出する最重要書類。記載事項は以下の通り:

| # | 記載事項 | 内容 |
|---|---------|------|
| 1 | 工事概要 | 工事名、工事場所、工期、請負金額 |
| 2 | 計画工程表 | バーチャート or ネットワーク |
| 3 | 現場組織表 | 現場代理人、主任(監理)技術者、専門技術者 |
| 4 | 指定機械 | 使用する建設機械の一覧 |
| 5 | 主要船舶・機械 | 海上工事の場合 |
| 6 | 施工方法 | 工種別の施工手順、仮設計画 |
| 7 | 施工管理計画 | 工程管理・出来形管理・品質管理・写真管理の方法 |
| 8 | 安全管理 | 安全管理体制、安全教育計画、第三者対策 |
| 9 | 緊急時の体制及び対応 | 災害時の連絡体制 |
| 10 | 交通管理 | 交通規制計画、誘導員配置 |
| 11 | 環境対策 | 騒音・振動・水質汚濁対策 |
| 12 | 現場作業環境の整備 | 仮設備、現場事務所 |
| 13 | 再生資源の利用の促進と建設副産物の適正処理方法 | リサイクル計画 |
| 14 | その他 | 特記仕様書で指定された事項 |

**自治体監督員のチェックポイント:**
- 共通仕様書の必須記載事項が網羅されているか
- 特記仕様書の条件が反映されているか
- 施工方法が設計図書の条件と整合しているか
- 品質管理・出来形管理の方法が施工管理基準に適合しているか

### 1-2. 出来形管理書類

**根拠:** 土木工事施工管理基準（国交省）、各自治体の施工管理基準

| 書類名 | 内容 | チェックポイント |
|--------|------|-----------------|
| 出来形管理図表 | 設計値と実測値の対比表 | 規格値内に収まっているか |
| 出来形管理図 | 断面図等に実測値を記入 | 測定箇所・頻度が基準を満たすか |
| 出来形測定結果一覧表 | 全測定値の一覧 | 管理基準値・規格値との照合 |
| 出来形管理総括表 | 工種別の管理結果まとめ | 全体の傾向、偏り |

**工種別の管理項目例:**

| 工種 | 測定項目 | 規格値例 |
|------|---------|---------|
| 土工（掘削） | 基準高▲、法長、幅 | -50mm |
| 路盤工 | 厚さ、幅、基準高▲ | 厚さ -25mm |
| コンクリート工 | 厚さ、幅、高さ、基準高▲ | -20mm |
| 舗装工 | 厚さ、幅、平坦性 | 厚さ -9mm（表層） |

### 1-4. 工事写真

**根拠:** 写真管理基準（案）（国交省）、デジタル写真管理情報基準

| 写真区分 | 内容 | 特に重要な場面 |
|---------|------|---------------|
| 着手前・完成写真 | 工事前後の対比 | 同一アングルでの撮影が必須 |
| 施工状況写真 | 施工中の各段階 | 不可視部分（埋設物等）は特に重要 |
| 安全管理写真 | 安全対策の実施状況 | 標識、仮設防護柵等 |
| 使用材料写真 | 搬入材料の確認 | 材料検収時 |
| 品質管理写真 | 試験実施状況 | スランプ試験、密度試験等 |
| 出来形管理写真 | 測定状況 | スケール当てによる寸法確認 |
| 災害写真 | 災害発生時の被害状況 | 緊急対応の記録 |

**自治体監督員のチェックポイント:**
- 黒板の記載内容（工事名、工種、測点、寸法等）が正確か
- 撮影箇所・頻度が写真管理基準を満たしているか
- 不可視部分の撮影漏れがないか
- デジタル写真の改ざん防止（JACIC信憑性確認）

## 2. 業務委託（設計・測量・調査）で確認する書類

### 2-1. 根拠となる仕様書

| 仕様書名 | 策定者 | 対象 |
|---------|--------|------|
| 設計業務等共通仕様書（案） | 国交省 | 土木設計業務 |
| 測量業務共通仕様書（案） | 国交省 | 測量業務 |
| 地質・土質調査業務共通仕様書（案） | 国交省 | 地質調査業務 |
| 各自治体の業務委託必携 | 各自治体 | 上記を自治体向けにカスタマイズ |

### 2-2. 照査報告書

**照査技術者の役割:**
- 業務の節目（中間・最終）で設計成果の照査を実施
- 設計条件の妥当性、計算の正確性、図面との整合性を確認
- 照査結果を照査報告書にまとめ、記名・押印して管理技術者に提出

**照査報告書の記載事項:**

| 項目 | 内容 |
|------|------|
| 設計条件の確認 | 適用基準、設計荷重、地盤条件等が正しいか |
| 設計計算の照査 | 計算過程・結果の正確性 |
| 図面の照査 | 設計計算書との整合、CAD基準への適合 |
| 数量計算の照査 | 数量計算書と図面の整合 |
| 報告書の照査 | 報告書の内容が業務目的に合致しているか |
| 照査結果の記録 | 赤黄チェック結果（赤=誤り、黄=確認済） |

### 2-3. 設計計算書

**自治体がチェックする項目:**

| チェック項目 | 具体例 |
|-------------|--------|
| 適用基準の妥当性 | 道路橋示方書、道路構造令、下水道施設計画・設計指針等 |
| 設計条件の設定 | 荷重条件、地盤定数、設計速度等 |
| 計算過程の正確性 | 断面力計算、安定計算、応力照査 |
| 使用ソフトの妥当性 | 認定ソフトの使用、入力条件の確認 |
| 結果の妥当性 | 過大・過小設計でないか |

### 2-4. 数量計算書

| チェック項目 | 内容 |
|-------------|------|
| 図面との整合 | 図面の寸法と数量計算の寸法が一致するか |
| 計算過程 | 数量算出の根拠が明確か |
| 単位・桁数 | 土木工事数量算出要領に準拠しているか |
| 工種区分 | 積算体系との整合 |

### 2-5. 図面（CAD基準への適合）

**CAD製図基準（国交省）での主なチェック項目:**

| チェック項目 | 基準内容 |
|-------------|---------|
| ファイル形式 | SXF(P21)形式 |
| レイヤ名 | 責任主体-図面オブジェクト-作図要素の命名規則 |
| 線色 | 背景色白で規定色 |
| 線種 | 実線、破線、一点鎖線等の使い分け |
| 文字 | MS明朝・MSゴシック、サイズ1.8mm以上 |
| 図面サイズ | A1を標準、輪郭線・表題欄 |
| 尺度 | 図面種別ごとの標準尺度 |
| 座標系 | 測量座標系との整合 |

## 3. 共通仕様書とは何か

### 3-1. 国土交通省の土木工事共通仕様書

**位置づけ:** 公共土木工事の施工に関する技術的事項を体系的に定めた書類。契約図書の一部。

**構成:**

| 編 | 内容 |
|----|------|
| 第1編 共通編 | 総則、材料、施工共通（適用範囲、用語、施工計画、安全対策等） |
| 第2編 材料編 | セメント、骨材、混和材料、鋼材、木材等の品質規格 |
| 第3編 土木工事共通編 | 土工、コンクリート工、基礎工等の施工方法・管理基準 |
| 第4〜11編 | 河川、道路、下水道、砂防、鋼橋、舗装、塗装等の専門工種 |

**改定サイクル:** 概ね毎年3〜4月に改定（令和7年4月が最新版）

### 3-2. 各自治体の独自仕様書

自治体は国交省版をベースに独自の仕様書を策定している:

| 名称パターン | 例 |
|-------------|-----|
| ○○市土木工事共通仕様書 | 神戸市土木請負工事必携 |
| ○○県土木工事共通仕様書 | 京都府土木工事共通仕様書（案） |
| ○○市土木工事書類作成マニュアル | 京都市工事関係書類作成マニュアル |
| ○○県施工管理基準 | 神奈川県施工管理基準 |

**国交省版との主な差異:**
- 地域特有の材料規格（寒冷地仕様、地場産材料の活用等）
- 独自の様式（打合せ簿、段階確認書等のフォーマット）
- 提出書類の簡素化 or 追加（自治体の規模・体制に応じて）
- 電子納品の対応レベル（義務化の範囲が異なる）
- 工事成績評定の配点・項目

### 3-3. 共通仕様書でチェックすべき主な項目カテゴリ

| カテゴリ | チェック内容 |
|---------|------------|
| 施工体制 | 技術者配置、下請体制、施工体制台帳 |
| 施工計画 | 施工計画書の記載事項の網羅性 |
| 材料 | 指定材料の品質規格、材料承認手続き |
| 施工方法 | 工種別の施工手順、仮設計画 |
| 施工管理 | 出来形管理基準値・規格値の遵守 |
| 品質管理 | 試験項目・頻度・基準値の遵守 |
| 安全管理 | 安全対策、第三者対策、交通管理 |
| 環境対策 | 騒音・振動・水質・廃棄物処理 |
| 写真管理 | 撮影項目・頻度・方法の遵守 |
| 電子納品 | フォルダ構成、ファイル命名、CAD基準 |

## 4. 自治体職員が書類チェックで困っていること

### 4-1. 技術職員不足による負担

- **約9割の「村」と約5割の「町」で土木部門職員が10人未満**（総務省調査）
- **約6割の「村」で土木部門職員が5人未満**
- 1人の監督員が複数工事を担当するのが常態化
- 監督業務に加えて窓口業務・住民対応・予算管理も兼務
- 技術系職員の採用試験で受験者ゼロのケースも発生（例: 和歌山県田辺市）
- 県からの技術職員派遣で対応するケースも（広域連携による支援）

### 4-2. チェック漏れのリスク

| リスク | 詳細 |
|--------|------|
| 出来形管理値の確認不足 | 規格値超過を見落とし、手戻り工事が発生 |
| 材料承認の形骸化 | 書類の内容を精査する時間がなく、形式的な確認にとどまる |
| 段階確認の省略 | 人手不足で立会いが間に合わず、写真確認のみで代替 |
| 設計図書の照査不足 | 設計と施工の齟齬を現場で初めて気づく |
| 検査時の確認不足 | 膨大な書類を限られた時間でチェックするため、サンプリングに頼る |

### 4-3. 書類の量と多様性

- 1件の工事で提出される書類は**数百〜数千ページ**に及ぶ
- 工種が多岐にわたるほど管理基準が増え、確認項目が爆発的に増加
- 書類の形式が受注者ごとにバラバラ（Excel、PDF、紙の混在）
- 複数工事を同時にチェックする必要がある
- 完成検査では書面検査と現場検査を限られた日程で実施
- **国の工事成績評定は「施工管理」「出来形及び出来ばえ」等の配点が高く、書類の品質が工事成績に直結**

### 4-4. 基準改定への追従

| 課題 | 詳細 |
|------|------|
| 頻繁な改定 | 国交省の共通仕様書は毎年改定。自治体版への反映にタイムラグ |
| 改定内容の把握 | 新旧対照表を確認し、影響範囲を特定する手間 |
| 電子納品基準の変更 | チェックシステムも改定に合わせてバージョンアップ（直近: 2026年4月改定） |
| 施工管理基準値の変更 | 工種別の規格値が変わると過去の判断基準が使えない |
| 複数基準の並存 | 国交省版と自治体版の差異管理が必要 |

## 5. 書類チェックの自動化に関する既存の取り組み・ツール

### 5-1. 国交省のASP（工事情報共有システム）

**概要:** 発注者と受注者がインターネットを通じて工事書類（打合せ簿、施工計画書、図面等）を共有・管理するクラウドシステム。

**主要ASP製品:**

| 製品名 | 提供者 | 特徴 |
|--------|--------|------|
| BasePage | 川田テクノシステム | シェアNo.1。書類の電子決裁、スケジュール管理 |
| 工事情報共有システム | 建設システム(KENTEM) | デキスパートとの連携 |
| BeingCollaboration | Being | BIM/CIM対応 |
| Calsmaster | 杉孝 | 帳票自動作成 |

**ASPで実現する効率化:**
- 打合せ簿の電子決裁（押印→電子承認）
- 提出書類の一元管理
- 電子納品データの自動整理
- 書類の提出・承認ステータスの可視化
- オンライン電子納品（ASPから直接納品）

**自治体への普及状況:**
- 国交省直轄工事ではASP利用が原則化（2020年〜）
- 都道府県・政令市レベルでは導入が進展中
- 市区町村レベルでは導入率が低い（予算・体制の制約）

### 5-2. 電子納品チェックシステム

**概要:** 国交省が無償提供する電子納品成果品の検証ツール。

**チェック内容:**
- フォルダ構成の検証
- ファイル命名規則の検証
- XML管理ファイルの妥当性検証
- CADデータのSXF形式チェック
- 文字コード・禁止文字のチェック
- 目視チェック支援機能（最新版で追加）

**最新動向:**
- 2026年4月にチェックシステム改定（電子納品に関する要領・基準の改定に対応）
- 国交省サイト（https://www.cals-ed.go.jp/）から無償ダウンロード可能

### 5-5. 自動化の空白地帯（CiviLinkのチャンス）

現状のソリューションを分析すると、以下の領域に自動化の空白がある:

| 空白領域 | 現状 | CiviLinkでの対応可能性 |
|---------|------|----------------------|
| **発注者側の書類チェック支援** | 受注者向けツールは豊富だが、発注者（自治体）向けのチェック支援ツールはほぼない | 自治体監督員向けのチェックリスト機能、図面照査機能 |
| **設計図書と施工記録の突合** | 人手で設計図面と出来形を照合している | 図面上にマークアップで出来形記録を紐付け |
| **基準改定の自動追従** | 改定のたびに人手でチェックリストを更新 | チェック項目マスタの自動更新 |
| **複数工事の横断的管理** | 工事ごとに別々のファイルで管理 | プロジェクト横断の図面・書類ダッシュボード |
| **維持管理段階の図面と施工記録の連携** | 施工完了後、図面と施工記録がバラバラになる | 図面に施工履歴・点検履歴を紐付けて蓄積 |

## まとめ: CiviLink自治体提案への示唆

1. **自治体の書類チェック業務は膨大かつ属人的** — 技術職員不足が深刻化する中、少人数で大量の書類を確認する必要があり、チェック漏れリスクが増大している

2. **受注者向けツールは充実しているが、発注者（自治体）向けは空白** — デキスパートやANDPAD等は受注者の書類作成を支援するが、自治体監督員のチェック業務を支援するツールはほぼ存在しない

3. **図面が多くの書類チェックの起点** — 出来形確認、段階確認、設計照査、数量照査のいずれも「図面」が起点。図面上にチェック結果・コメント・写真を紐付けられるCiviLinkの機能は、書類チェック業務との親和性が高い

4. **電子納品・CAD基準への適合チェックはニーズが高い** — 特に市区町村レベルではチェック体制が脆弱。CiviLinkがCADデータの閲覧・チェックを容易にすることで差別化可能

5. **AI照査の流れは加速中** — 大手ゼネコン・コンサルは自社でAI照査を開発。中小自治体・中小建設会社にはまだ届いていない。CiviLinkのプラットフォーム上にAI照査機能を載せれば、中小向けの差別化ポイントになりうる
