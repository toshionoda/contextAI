## 2. 書式の共通点（パターン分析）

### 2.1 数量計算書／総括表の階層構造

**全サンプルで「工種 → 種別 → 細別 → 規格 → 単位 → 数量」の4-6階層構造が共通**

| サンプル | 列構成（数量総括表） |
|---|---|
| 福岡市 | 工種(L2)・種別(L3)・細別(L4)・規格・単位・数量・備考 |
| 横瀬町 | 工種・種別・細別・規格・数量・設計数量・単位・数位・備考 |
| 宮城県道路公社 | 計上数量総括・本体付属別総括・章別計算書 |
| 国交省 道路トンネル | §1数量総括表 / 1-1数量総括表 / 1-2数量集計表 / §2数量延長調書 / §3トンネル掘削工 / §4支保工 ... |
| 沖縄県企業局 | 工種・種別・細別/名称・単位・数量・単価・金額・摘要 |

→ **核となる構造（工種＞種別＞細別 + 規格・単位・数量）は完全に共通**

### 2.2 図面構成の共通点

**全サンプルで以下の図面が同じ構造で存在**:
- 位置図（縮尺1/2,500〜1/5,000）
- 平面図（縮尺1/300〜1/500、A1版）
- 縦断図（V=1/100, H=1/300〜1/500）
- 横断図（縮尺1/100〜1/200）
- 詳細図（必要に応じ）

**福岡市と名古屋市の標準が記載項目レベルで完全一致**:
- 新設管きょの記載項目: 施工種別、排水区分、管種、断面形状、管径、勾配、延⾧、上流管底高、下流管底高、耐震種別
- 新設マンホール: 番号、型式、内寸、外寸、設置深、地盤高、オフセット

### 2.3 文書セット構成の共通点

```
[図面一式（CAD/PDF）]
  ↓
[数量計算書（路線・章別計算過程）]
  ↓
[数量総括表（工種階層集計）]
  ↓
[金抜き設計書（単価適用・参考資料）]
  ↓
[入札公告（金入り設計書は非公開）]
```

→ **すべての自治体・公共発注体でこの流れが共通**

### 2.4 JS（日本下水道事業団）標準による全国統一

**JSの「下水道施設CAD製図基準」（ZH004-00-18-A、令和元年11月一部改正版、97p）が決定的**:

#### レイヤ命名規則（4階層・全国統一）
```
[責任主体]-[図面オブジェクト]-[作図要素]-[ユーザ定義領域]
例: D-TTL-FRME / D-STR-STR4 / D-DCR-HCH1
```

| 階層 | 内容 | 例 |
|---|---|---|
| ①責任主体 | S（測量）・D（設計）・C（施工）・M（維持管理）の1文字 | D |
| ②図面オブジェクト | 9種類（TTL=図枠, STR=主構造物, DCR=装飾, など）の3文字 | STR |
| ③作図要素 | 1-5文字（HTXT=旗上げ等） | STR4 |
| ④ユーザ定義領域 | 244文字以内 | （任意） |

#### ファイル命名規則・図面種別CODE BOOKも標準化済み

#### カバレッジ
- 共通・土木・建築・建築機械・建築電気・機械・電気の7カテゴリ
- 下水道基本構想・公共下水道全体計画・下水道法事業認可設計・都市計画決定・都市計画事業認可の5フェーズ
- 水コン協CAD基準との対比表もあり

→ **下水道工事のCAD図面は、レイヤ名で意味が機械的に把握可能な状態に標準化されている = AI自動化の決定的な追い風**

### 2.5 国交省 i-Construction 2.0 政策による標準化加速

- 2026年マイルストーンで3Dモデル+設計支援ソフトの数量を直接積算で使える流れ
- BIM/CIM全面適用の計画あり
- 標準書式・コード体系がさらに統一される方向

## 3. 書式の差異（自治体間バリエーション）

### 3.1 単価コード体系（自治体ごとに独自）

| 自治体・発注体 | コード体系の例 |
|---|---|
| 福岡市 | B07GH-XX-XX-G0（設計書番号）、PG/CB/ZD/PD（単価コード）、第1号代価表… |
| JS（日本下水道事業団） | CF001-01-23-A（土木工事 積算基準・標準歩掛 令和5年度） |
| 国交省 様式63-2号 | 全国一般様式（A4判） |
| 沖縄県企業局 | 様式１・様式１０ |

→ **自治体ごとに独自だが、構造は同じ「単価コード→規格→単位→数量」のマッピング** = 設定切替で対応可能

### 3.2 列構成の細かい差異

| 自治体 | 数量関連列の構成 |
|---|---|
| 福岡市 | 数量・備考（2列） |
| 横瀬町 | 数量・設計数量・数位・備考（4列：丸めの厳密化） |
| 沖縄県企業局 | 数量・単価・金額（3列：金抜きでも単価欄あり） |

→ 列のラッピング・順序の違い。**テンプレ設定で吸収可能**

### 3.3 工種分類の細部

| 工種カテゴリ | 福岡市 | 名古屋市標準 | 共通性 |
|---|---|---|---|
| 開削工事（下水道築造） | あり | あり | 同一 |
| 更生工事（下水道改築） | あり | あり | 同一 |
| 推進・シールド工事 | 該当なし（小規模） | あり | 共通体系 |
| 取付管工事 | あり | あり | 同一 |
| 付属工事（腹付管・圧送管等） | 一部あり | あり | 共通体系 |

→ **政令市・JSレベルで分類体系が統一**。福岡市の数量総括表 4分類（管更生工 / 管布設工 / 取付管・ます・人孔工 / 付帯工）はこの中に綺麗にマッピングされる

## 4. AI自動化への影響評価

### 4.1 自動化の親和性

| 観点 | 評価 | 根拠 |
|---|---|---|
| **書式パターン共通性** | **★★★** | 工種階層・図面構成・記載項目がJS標準で全国統一 |
| **CAD図面の構造化** | **★★★** | JSレイヤ命名規則によりレイヤ名から意味が機械的把握可能 |
| **テンプレ吸収可能な差異** | **★★★** | 単価コード・列構成は自治体ごとに独自だが、設定で対応可能な範囲 |
| **政策的標準化トレンド** | **★★★** | i-Construction 2.0で標準化がさらに進行中 |

### 4.2 「同じパターン」前提の成立度

**結論: 強く成立する**

- 下水道工事の図面・数量計算書は **JS標準＋政令市標準** で全国統一されている
- 福岡市の書式は決して特殊ではなく、**全国共通テンプレートの一実装**
- 自治体ごとの差異は「単価コード体系」と「列の細かい順序」程度
- これは社内アセット5（rd-poc-drawing-calc-matcher）で **構造別の knowledge.md** で吸収する設計と完全に整合する
  - 鉄筋では U型擁壁 / 橋脚躯体 / 橋脚基礎 / 橋脚梁 / RC上部工 の5構造で対応済
  - 下水道では 管更生 / 開削 / 推進・シールド / 取付管 / 付帯 の5工種で同様に対応可能

### 4.3 残るリスク（構造的差異）

以下の場合は別アプローチが必要：

| リスク | 影響 | 対応案 |
|---|---|---|
| 紙図面（旧案件） | OCR精度低下 | 対象外（i-Construction 2.0で電子化加速） |
| 中小自治体の独自書式 | 政令市標準と異なる場合あり | テンプレ追加で対応（コスト増） |
| 工事種別による差異 | 推進・シールド・処理場で記載項目が大きく異なる | フェーズ展開で対応 |
| 工事規模による粒度差 | 大規模（トンネル419p） vs 小規模（数量計算書数十p） | 段階的に対応 |

### 5.5.4 自動化AIから見た「同じパターン」前提の検証結果

**結論: 強く成立する**

- **構造を決める要素（レイヤ・記号・規格部品・図面オブジェクト分類）は全国統一**
- **都市独自の要素（管理項目・単価コード）は表面的なメタデータ層**
- 福岡市で作ったMVPの **コアロジックは政令市レベルで85-90%再利用可能**、残り10-15%は「テンプレ設定で吸収」する設計が成立
- 国の i-Construction 2.0 政策（2026年マイルストーン）でさらなる収斂が見込まれる

**東京都の事例が象徴的**: 平成18年から続けた独自「管きょ設計CAD」を **2024年に国際標準SXF(P21)に統合**。政令市レベルでは独自仕様を維持するコストの方が高いと判断されている。

### 5.5.5 横浜市の特異点（注意）

- 横浜市は「**JSWAS全面参照**」かつ「国交省CAD製図基準採用」で、自治体としてはトップクラスの標準準拠
- 一方で **管理項目の追加カラム**は独自で、横浜市版テンプレで吸収する必要あり
- 改定履歴25回（昭和51年から令和5年）= **継続的にメンテナンスされている運用ドキュメント**
- 自治体パッケージ展開時、横浜市は早期ターゲット（書式互換性が高く、運用が成熟）

| 残領域 | 取得方法 |

## 7. 結論

1. **「同じパターン」前提は強く成立**: JS標準＋国交省CAD製図基準＋JSWAS規格＋政令市の国基準準拠で全国統一
2. **横浜市の宣言が制度的根拠**: 「国基準を採用するのが原則」「多くの自治体が同様」を政令市が明文化
3. **東京都の事例が象徴的**: 独自仕様「管きょ設計CAD」を2024年4月にSXF(P21)国際標準へ統合 = 独自仕様を捨てる流れ
4. **福岡市は標準的な政令市の代表例**: ここで作ったMVPは他政令市に85-90%再利用可能
5. **差は表面的（管理項目・単価コード・表題欄）**: テンプレ設定で吸収可能、事業性試算の想定範囲内
6. **JS（日本下水道事業団）が顧客になる可能性は最強の横展開シナリオ**: 全国の自治体発注を受託している
7. **社内アセット（特にrd-poc-drawing-calc-matcher）の構造別knowledge.md設計と完全に整合**: 鉄筋5構造→下水道5工種への置き換えで対応可能
