## 2. なぜ今、施工会社向けメニューが必要か

### 佐賀市の状況（2026-04-09キックオフでの確認事項）
- 伊藤忠商事と佐賀市の**包括委託契約が1年延期**
- この1年を**POCの場として活用**することで合意済み
- 市の要望：CiviLinkは「データプラットフォーム」→「**地場建設業界のDX普及推進**」へ位置付け変更
- 包括委託スキーム上、**伊藤忠グループだけでは業務遂行不可能** → 地元施工会社の巻き込みが不可欠

### 4/20佐賀市訪問
- 伊藤忠が佐賀市デジタル推進課を訪問
- マルメは**DXパートナーとして正式紹介**される
- パシコン・マップ系企業と並んで紹介される予定
- → **その場で配る/見せるメニュー資料が必要**

## 4. CiviLinkのポジショニング（説明用）

### 既存ツールとの棲み分け
```
施工管理（ANDPAD等）  ← 競合しない
       ↕
CiviLink              ← 「図面起点の照査・管理」に特化
       ↕
台帳管理（Genavis等）  ← 補完関係
```

### 3層モデル（羅針盤2026より）
| Layer | 内容 | 2026年目標 |
|-------|------|-----------|
| **Layer 1** | SaaS基盤（差分比較・版管理・指摘管理） | PMF確立 |
| **Layer 2** | AI業務代行（照査・積算・帳票AI） | PoC→有償化 |
| **Layer 3** | FDE伴走（現場エンジニアによる定着支援） | 採用促進 |
