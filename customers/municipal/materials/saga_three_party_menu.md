# 佐賀市向け 3者連携メニュー：自治体 × 補修設計会社 × 地場建設会社

**作成日:** 2026-04-14
**対象:** 佐賀市デジタル推進課、佐賀市契約の補修設計コンサル、Cランク以下の地場建設会社
**根拠資料:** 佐賀市橋りょう長寿命化修繕計画（令和4年策定）、令和8年度発注見通し

---

## エグゼクティブサマリ

橋梁長寿命化計画は、**自治体（発注者）→ 補修設計会社 → 地場建設会社（施工者）→ 自治体（維持管理）** のサイクルで動いている。各プレイヤーが個別最適でDXツールを導入しても、データが繋がらないと**長寿命化計画の運用そのものが効率化されない**。

本提案では、**3者それぞれに刺さる個別メニュー**を提示しつつ、**CiviLinkがその3者を横串で繋ぐデータ基盤**として機能する構造を示す。

### 3者連携の全体像

```
【佐賀市】2,842橋の管理  900橋を自前点検
   │
   ├─ 点検データ → 補修要否判定 → ★補修設計発注★ ─→ 【補修設計会社】
   │                                                       │
   │                                                       ↓
   │                                              ★補修設計成果★
   │                                                       │
   ├─ 設計照査 → 補修工事発注 ────────────────→ 【地場建設会社】
   │                                                       │
   │                                                       ↓
   ├─ 工事監督・段階確認 ←─────────── ★工事打合簿・写真★
   │
   └─ 完成図書受領 → 次回点検データの起点に
```

---

## なぜ「3者連携」なのか

### 単一ステークホルダーへの提案では不十分

| 単独提案の限界 | 3者連携で得られる効果 |
|--------------|---------------------|
| 自治体だけにDXを入れても、設計会社・施工会社の成果物がアナログだとデータが活きない | 点検→設計→施工→次回点検のデータが「川のように流れる」 |
| 設計会社・施工会社だけにDXを入れても、自治体側がアナログだと結局紙で出す必要がある | 自治体監督員のチェック業務も同時に効率化 |
| 個別最適のツール導入は重複・断絶を生む | CiviLinkが「共通言語（図面ベース）」になる |

### 長寿命化計画の取組メニューと一致

佐賀市橋りょう長寿命化修繕計画の基本方針には以下が明記されている：
- ✅ **データ管理方法の検討** → CiviLinkのコア機能
- ✅ **職員が円滑に維持管理できる体制構築** → FDE伴走
- ✅ **技術継承の仕組みの構築** → AI/データ蓄積（SoK）
- ✅ **新技術活用方針（NETIS等）** → 提案メニューはこの文脈に乗る

## ① 自治体向けメニュー（佐賀市デジタル推進課・道路課・建設事務所）

### 業務サイクルと現状の課題

```
2,842橋を管理 → 900橋を自前点検 → 損傷記録 → 補修要否判定 → 補修設計発注
   ↓                   ↓               ↓             ↓
[紙台帳・GIS]      [紙の点検野帳]   [Excel手入力]   [仕様書手書き]
                   [写真バラバラ]   [属人的判断]    [メール添付発注]
   ↓
補修工事発注 → 監督・段階確認 → 完成図書受領 → 次回点検の起点
   ↓             ↓                  ↓
[FAX往復]    [現場立会・紙確認]    [紙ファイル保管]
```

### メニュー（自治体）

| # | メニュー | 自治体メリット | 想定効果 |
|---|---------|--------------|---------|
| **A1** | 自前点検データのデジタル化・一元管理 | 900橋の点検結果を**橋梁単位で蓄積**。担当者異動でも継承可能 | 検索時間ゼロ化、技術継承 |
| **A2** | **★補修設計発注の効率化** | 過去仕様書テンプレ＋点検データから**仕様書ドラフトAI生成**。発注業務工数を削減 | 仕様書作成時間 **50%削減** |
| **A3** | 補修設計成果の照査支援 | 設計コンサルから受領した図面・数量計算書を**CiviLink上で照査**。指摘・回答履歴が残る | 紙のやり取り根絶 |
| **A4** | 補修工事の監督業務支援 | 段階確認・材料承認・打合簿を**CiviLink上で完結**。立会移動時間を削減 | 監督員1人あたり**月20時間削減** |
| **A5** | 長寿命化計画運用ダッシュボード | 2,842橋の健全度・補修履歴・予算消化を**1画面で可視化**。優先順位の判断材料に | 計画見直しサイクルの短縮 |
| **A6** | 課横断の発注連携基盤 | 道路整備課・南部/北部建設事務所・森林整備課で**発注情報・実績を共有** | 重複発注防止、調整工数削減 |
| **A7** | 技術継承プラットフォーム | ベテラン職員の判断パターン・補修事例を**AIが学習**。若手の意思決定支援 | 技術継承の仕組み化 |

### 数字で示すインパクト

長寿命化計画10年費用：**設計費9.8億 + 修繕費21.2億 + 点検費10億 = 41億円**
- 仮に**設計発注の効率化で3%削減** = **約3,000万円/10年**
- 仮に**監督業務効率化で工事5%削減** = **約1億円/10年**
- 計画コスト縮減目標 70億円（50年）に対する寄与

## 3者を貫くCiviLinkの役割

### 「データの川」を作る

```
            ┌────────────────────────────────┐
            │      CiviLink（データ基盤）       │
            │                                  │
   自治体 ──┤  点検データ → 設計データ → 施工データ ──┐
            │      ↑                         ↓        │
            │      └────  完成図書 ────────  ┘        │
            │                                          │
            │  + チャット / 写真アプリ / Webカメラ      │
            │    / クラウド / 電子契約 を横串連携       │
            └──────────────────────────────────┘
                ↑                ↑              ↑
            自治体          補修設計会社    地場建設会社
```

### CiviLink独自の3つの差別化

1. **3者全員が同じ図面を見る** — 紙の図面→PDF→メールで散発的に共有していた状態が、図面ベースの「単一の真実」に
2. **データが流れる** — 点検データ（自治体）→ 設計入力（コンサル）→ 施工基準（建設会社）→ 完成図書（自治体に戻る）
3. **既存ツールを束ねる** — CiviLink単独では完結せず、各社が使い慣れたツール（蔵衛門・LINE WORKS等）と連携

## マルメのポジショニング

### 単なるツール提供者ではなく「3者を繋ぐDXコンサル」

| 役割 | 内容 |
|------|------|
| **DXコンサル** | 自治体・設計会社・建設会社それぞれの現場に入り、業務分析から最適なツール選定 |
| **既存ツールパッケージ提案** | 1から開発せず、市場にある良いツールを組み合わせる |
| **CiviLink提供** | 3者を繋ぐデータ基盤を提供 |
| **補助金申請サポート** | 補助金活用で実質負担を最小化 |
| **FDE伴走** | エンジニアが現場に常駐に近い形で定着支援 |
| **データ蓄積→AI開発** | 蓄積したデータをもとに将来的にAI機能を開発（Layer 2） |

## POC構成の提案

### Phase 1: 「3者シングル橋梁POC」（2026年5月〜10月）

**対象**: 1橋を選定し、3者全員が参加して一気通貫の流れを実証

| 月 | 自治体 | 設計会社 | 建設会社 |
|---|-------|---------|---------|
| 5月 | 既存点検データ投入 | - | - |
| 6月 | 補修設計発注（CiviLink上） | 受注・点検データ受領 | - |
| 7-8月 | 設計照査 | 補修設計実施 | - |
| 9月 | 工事発注 | 設計完了・引渡 | 受注・施工計画 |
| 10月 | 監督・段階確認 | - | 工事実施・写真連携 |

### Phase 2: 拡大POC（2026年11月〜2027年3月）

**対象**: 5〜10橋に拡大、3者の体制を恒常化

### Phase 3: 包括委託本格運用（2027年4月〜）

3者連携モデルを包括委託の標準業務フローとして定着
