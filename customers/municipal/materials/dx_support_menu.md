# 地場建設会社向け DX支援メニュー

**作成日:** 2026-04-09
**目的:** 4/20佐賀市訪問にて、マルメがDXパートナーとして提供できるメニューを紹介する
**対象:** 佐賀市の包括委託に参画する地場建設会社（中小規模）
**ステータス:** ドラフト（4/14までに確定）

---

## 背景: 地場建設会社が直面する3つの壁

### 1. 2024年問題 → 残業できない時代
- 2024年4月から時間外労働の上限規制が建設業にも適用（月45時間・年360時間が原則）
- 現場が終わった後の書類作成・写真整理・図面管理が残業の温床

### 2. 技術者の高齢化 → ベテランがいなくなる
- 建設業就業者の37%が55歳以上、29歳以下はわずか12%
- 2035年には技術者約3.2万人、技能労働者約27.8万人が不足する予測
- 「あの図面どこにあるか知ってるのは○○さんだけ」が日常

### 3. i-Construction 2.0 → デジタル対応が必須に
- **2026年度から舗装工・地盤改良工でICT施工が原則化**（佐賀市の道路工事に直撃）
- 電子納品・遠隔臨場の対応範囲も拡大中
- 自治体公共工事でのデジタル対応力が受注競争力に直結

## 1. 図面のクラウド一元管理

**課題:** 竣工図・道路台帳・施工図面が事務所のキャビネット、個人PC、USBメモリにバラバラ。必要な図面を探すだけで時間がかかる。担当者が不在だと見つからない。

**CiviLinkでできること:**
- 全ての図面をクラウドにアップロード、フォルダで整理
- ファイル名・メタデータで検索、必要な図面を即座に特定
- 誰でも・どこからでもアクセス可能（PC・タブレット・スマホ）
- アクセス権限設定で社内外の共有範囲をコントロール

**包括委託での活用イメージ:**
- 受託エリアの道路台帳・竣工図面をCiviLinkに集約
- JV構成企業全社が同じ図面にアクセス → 事業所間の図面輸送が不要に
- パトロール前に該当区間の図面をタブレットで確認してから出動

**地場建設会社にとってのメリット:**
- 「○○さんしか知らない」状態の解消 → 属人化リスクの低減
- 図面を探す時間の大幅削減（倉庫に行かなくていい）
- 災害・緊急時にも図面にすぐアクセスできる

---

## 2. 現場での図面閲覧

**課題:** 現場に紙図面を持ち出すと汚損・紛失のリスク。A1サイズの図面は持ち運びが大変。修正があっても古い版を持ったまま作業してしまう。

**CiviLinkでできること:**
- タブレット・スマホからPDF図面をベクター表示で閲覧
- ピンチイン/アウトで拡大縮小しても文字・線がくっきり鮮明
- オフライン対応可能（事前ダウンロード）
- 常に最新版が表示される

**包括委託での活用イメージ:**
- 道路パトロールでタブレット片手に損傷箇所を図面上で確認
- 補修現場で施工図面を見ながら作業、赤入れもその場で
- 発注者との現場打ち合わせで図面を画面共有

**地場建設会社にとってのメリット:**
- 紙図面の印刷・コピー・持ち出し管理が不要に
- 現場と事務所の往復削減（「図面を取りに戻る」がなくなる）
- 古い版の図面で施工するミスを防止

---

## 3. デジタル照査・指摘管理

**課題:** 図面チェックは赤ペンで紙に書き込み、それをスキャンしてメールで送る。やり取りが煩雑で、指摘の対応漏れが発生する。

**CiviLinkでできること:**
- 図面上の任意の箇所にコメント・マークアップを追加
- 指摘ごとにスレッド形式で議論・回答・承認
- 指摘の対応状況を一覧で管理（未対応・対応中・完了）
- 写真の添付も可能

**包括委託での活用イメージ:**
- パトロールで発見した損傷箇所を図面上にマーキング + 写真添付
- 補修計画の照査を図面上で実施（赤入れ→回答→承認のサイクル）
- 指摘履歴がそのまま品質記録として蓄積される

**地場建設会社にとってのメリット:**
- 赤ペン→スキャン→メールの手間がゼロに
- 指摘の見落とし・対応漏れを防止（一覧管理）
- 過去の指摘が検索可能 → 同じ失敗を繰り返さない

## 4. 補修工事の承認ワークフロー

**課題:** 包括委託では簡易な補修は事業者判断で実施するが、一定規模以上は自治体の承認が必要。紙ベースだと承認に時間がかかり、補修対応が遅延する。

**CiviLinkでできること:**
- 補修箇所を図面上で特定し、補修内容をコメントで記載
- 自治体担当者をプロジェクトに招待し、CiviLink上で確認・承認
- 補修前後の図面・写真を紐付けて記録
- 承認の履歴がすべて残る（監査対応）

**包括委託での活用イメージ:**

```
[パトロールで損傷発見]
    ↓ 図面上にマーキング + 写真
[施工会社が補修計画作成]
    ↓ CiviLink上で共有
[自治体担当者が照査・承認]
    ↓ コメントで承認 or 差し戻し
[補修実施 → 完了報告]
    ↓ 補修後の写真・記録を追加
[履歴として蓄積]
```

**地場建設会社にとってのメリット:**
- 承認待ちの時間短縮（メール・FAXの往復がなくなる）
- 自治体とのやり取りが図面ベースで明確になる
- 「言った・言わない」がなくなる（全て記録される）

## 5. 図面バージョン管理・履歴管理

**課題:** 補修のたびに図面が更新されるが、「最新版」「修正版v2」「最終版（本当）」のようなファイル名管理が横行。どれが正しい図面か分からなくなる。

**CiviLinkでできること:**
- 図面のバージョンを自動で履歴管理
- いつ・誰が・何を変更したかが記録される
- 過去のバージョンにいつでも戻れる
- 最新版が常に明確

**包括委託での活用イメージ:**
- 補修前後の図面をバージョンとして管理 → 道路の変遷記録に
- 年次報告で「この1年でどの区間をどう補修したか」を図面履歴で可視化
- 次期包括委託への引き継ぎ資料が自動的に蓄積される

**地場建設会社にとってのメリット:**
- 「最新版どれ？」問題の完全解消
- 旧版で施工してしまうリスクを排除
- 過去の施工履歴が資産として残る（ノウハウの蓄積）

## 6. 書類の基準適合チェック支援

**課題:** 自治体の監督員は、共通仕様書・施工管理基準に基づいて膨大な書類をチェックしなければならない。1件の工事で数百〜数千ページ。しかし**発注者（自治体）のチェック業務を支援するツールはほぼ存在しない**。

**CiviLinkでできること（現行機能）:**
- 図面上にマークアップ・コメントで照査結果を記録（赤黄チェックのデジタル化）
- 設計変更時のバージョン差分を可視化 →「何が変わったか」を即座に把握
- 段階確認の記録を図面に紐付けて管理
- 指摘の対応状況を一覧管理（チェック漏れ防止）

**将来的に実現可能なこと:**
- 共通仕様書の確認項目をテンプレート化し、チェックリストとして提供
- 出来形管理値の規格値自動照合（入力値 vs 基準値のOK/NG判定）
- 図面⇔数量計算書の整合性チェック（AI+OCR）
- 共通仕様書・設計基準に基づくAI照査アシスタント

**包括委託での活用イメージ:**
- パトロール報告書の確認を図面ベースで効率化
- 補修実施報告書のチェック → 指摘 → 是正の一気通貫
- 年次報告の基礎データが自動蓄積

**自治体にとってのメリット:**
- 監督員のチェック負担軽減（特に複数工事の同時管理時）
- チェック漏れリスクの低減
- 確認記録が自動的に監査証跡として残る
- 基準改定時にテンプレートを更新するだけで全工事に反映

**詳細:** [書類自動チェックシステム提案](document_auto_check_proposal.md)

## 7. AI橋梁点検・損傷検出

**課題:** 橋梁の近接目視点検には橋梁点検車（BT車）、交通規制員、点検技術者2〜3名が必要。1橋あたり2〜3日かかり、コストも高い。地場建設会社にとって橋梁点検は負担が大きい。

**推奨ソリューション:**

| ツール | 費用感 | 概要 |
|--------|--------|------|
| **ひびみっけ（富士フイルム）** | 月額数万円〜 | スマホ撮影→AI解析。0.1mmのひび割れ検出、損傷図自動生成。**導入ハードル最低** |
| ドローン点検（外注） | 1橋30〜100万円 | 自社でドローンを持たず外注。最も手軽 |
| 赤外線サーモグラフィ | レンタル1日5〜10万円 | コンクリート内部の浮き・剥離を非接触検出 |
| 水中ROV（レンタル） | 1日5〜10万円 | 橋脚の水中部点検。洗堀状況確認 |

**導入効果（経団連提言より）:**
- **人件費7割削減**: BT車+交通規制+複数名 → ドローン1名+補助1名
- **作業時間4割削減**: 2-3日/橋 → 半日〜1日 + AI損傷図自動生成

**包括委託での活用:**
- 受託区間の全橋梁を効率的に巡回点検
- AI解析結果をそのまま点検調書に活用
- 蓄積データで劣化傾向を分析 → 予防保全計画に反映

**マルメの役割:** ツール選定のアドバイス、CiviLinkとのデータ連携（AI解析結果の図面紐付け）

## 8. ドローン点検・測量

**課題:** 2026年度から舗装工のICT施工が原則化。起工測量・出来形計測で3Dデータが必要になるが、中小建設会社にはノウハウも機材もない。

**推奨ソリューション:**

| 段階 | 選択肢 | 費用感 |
|------|--------|--------|
| **まず外注** | 撮影+データ処理を外注 | 1現場15〜50万円 |
| **自社導入（入門）** | DJI Mavic 3 Enterprise RTK | 機体60〜80万円 |
| **自社導入（本格）** | DJI Matrice 350 RTK + Zenmuse P1 | 250〜300万円 |
| **LiDAR（樹木下）** | DJI Zenmuse L2 | 150〜200万円（センサー） |

**写真測量 vs LiDAR:**

| | 写真測量 | LiDAR |
|--|---------|-------|
| コスト | 低い（100〜300万円） | 高い（300〜1,000万円+） |
| 植生下の計測 | 不可 | 可能 |
| 推奨場面 | 造成・舗装・開けた現場 | 法面・山間部・河川堤防 |

→ 中小は**写真測量から開始**が現実的。

**必要な資格:**
- 二等無人航空機操縦者技能証明（取得費用20〜35万円、4〜10日間）
- 外注利用なら資格不要

**点群処理ソフト:**

| ソフト | 価格 | 特徴 |
|--------|------|------|
| DJI Terra | 15〜60万円/年 | DJI機体と親和性◎。入門に最適 |
| Metashape | 6〜35万円（買切） | コスパ最強 |
| **Terra Mapper** | 40〜80万円/年 | 福岡のテラドローン社。九州サポート手厚い |
| **TREND-POINT** | 50〜100万円 | 福井コンピュータ。i-Con対応のスタンダード |

## 9. GNSS/RTK測量の導入

**課題:** 従来のTS（トータルステーション）測量は2人作業で時間がかかる。i-Construction対応の起工測量・出来形測量にはcm精度のリアルタイム測位が必要。

**推奨ソリューション:**

| 区分 | 推奨 | 費用感 |
|------|------|--------|
| **補正サービス** | ジェノバVRS or docomo IoT GNSS | 月3〜15万円 |
| **受信機（コスパ重視）** | Topcon HiPer VR / Sokkia GCX3 | 150〜300万円 |
| **受信機（高機能）** | Trimble R12i（チルト補正付き） | 200〜350万円 |

**導入効果:**
- 起工・出来形測量の工数 **約40〜50%削減**
- 1人での計測が可能（TS測量は2人必要）
- 基準局設置不要（ネットワーク型RTK利用時）

**i-Construction対応の最優先投資**: 2026年度の舗装工原則化に対応するには、GNSS/RTKが最低限必要。**これが全ての起点**。

## 10. 2D-MG（マシンガイダンス）導入

**課題:** ICT施工の原則化が進むが、ICT建機の新規購入は数千万円で中小には無理。既存重機を活かしたい。

**推奨ソリューション:**

| 製品 | 価格 | 特徴 |
|------|------|------|
| **Topcon MC-Mobile** | 150〜350万円 | 入門に最適。バックホウ対応 |
| Trimble Earthworks | 200〜400万円 | 2D/3D両対応 |
| SMART CONSTRUCTION Retrofit | 150〜300万円 | コマツ以外にも対応 |

**2D-MGとは:** 設計データとバケット位置をモニター表示してオペレータを誘導するシステム。自動制御（MC）と異なり**既存重機に後付け**でき、中小のICT施工の入り口。

**導入効果:**
- 小規模管渠掘削で従来比 **約54%の省力化** 達成事例あり（国交省モデル事業）
- 丁張り設置が不要 or 大幅削減
- 施工精度の向上

## 11. 紙図面のデジタル化支援

**課題:** 昭和〜平成初期に作られたインフラの図面は紙のまま倉庫に保管されている。デジタル化したいが、何から手をつけていいか分からない。

**マルメが提供できること:**
- 紙→PDF化の最適なスキャン方法・設定のアドバイス
- PDF品質の確認（CiviLinkのベクター表示に適した形式かどうか）
- DocuWorks（XDW）→ PDF変換の知見提供（JRC等での豊富な実績）
- デジタル化の優先順位設計（全部一気にやらず、使用頻度の高い図面から段階的に）

**包括委託での活用イメージ:**
- 受託エリアの主要道路の竣工図面を優先的にデジタル化
- パトロール頻度の高い区間の図面から着手
- デジタル化と同時にCiviLinkにアップロード → すぐに現場で使える状態に

**注: これはCiviLinkの機能ではなく、マルメのコンサルティングサービスとして提供**

## CiviLinkのポジショニング

### 既存ツールとの棲み分け

```
施工管理（ANDPAD, 蔵衛門等）  ← 施工段階の現場管理（写真・工程・受発注）
         ↕ 競合しない
CiviLink                      ← 維持管理段階の図面活用（閲覧・照査・バージョン管理）
         ↕ 補完関係
台帳管理（Genavis等）          ← インフラ資産全体の台帳・AI診断
```

- CiviLinkは施工管理アプリ（ANDPAD等）と**競合しない**
- 台帳管理システム（包括委託で導入されることが多い）とも**補完関係**
- 「図面を見る・チェックする・管理する」に特化した唯一のポジション

### 導入の段階イメージ

```
Step 1（即日〜）    図面をアップロード、現場でタブレット閲覧
Step 2（1ヶ月〜）   照査・コメント機能で社内レビューをデジタル化
Step 3（3ヶ月〜）   自治体担当者も招待、承認ワークフローを構築
Step 4（6ヶ月〜）   包括委託JV全体の図面基盤として定着
```

## 提案時の注意点

1. **「DX」という言葉を使いすぎない** — 中小建設会社は横文字に抵抗がある。「図面をスマホで見られるようにする」レベルの具体性で話す
2. **大手ゼネコンの事例を出さない** — 「うちとは規模が違う」と距離を置かれる。同規模の地場建設会社の事例が理想
3. **段階的導入を強調** — 「まず図面を見るところから」。全機能を一度に説明しない
4. **コストの話を先に出す** — 補助金で実質負担がこれだけ、と具体的に示す
5. **現場の人に刺さる言葉** — 「倉庫に図面取りに行かなくていい」「雨の日に紙図面が濡れない」
