# 自治体向け 書類自動チェックシステム提案

**作成日:** 2026-04-09
**目的:** 自治体が公共工事・業務委託で確認すべき書類を整理し、CiviLinkを活用した自動チェックの可能性を提案する
**ステータス:** ドラフト

---

## 1. 自治体が確認しなければならない書類の全体像

### 1-1. 公共工事（施工段階）

自治体の監督員は、共通仕様書・施工管理基準に基づいて以下の書類を確認する。
1件の工事で**数百〜数千ページ**に及ぶ。

| カテゴリ | 主な書類 | チェック基準 | 確認タイミング |
|---------|---------|-------------|---------------|
| **施工計画書** | 工事概要、施工方法、安全管理計画、品質管理計画等（14項目） | 土木工事共通仕様書 | 着手前 |
| **出来形管理** | 出来形管理図表、管理図、測定結果一覧表 | 施工管理基準（工種別の規格値） | 施工中〜完了 |
| **品質管理** | 品質管理図表、試験成績表、配合報告書、材料試験成績書 | JIS規格、施工管理基準 | 施工中 |
| **工事写真** | 着手前/完成、施工状況、安全管理、使用材料、出来形管理 | 写真管理基準（案） | 全期間 |
| **安全管理** | 安全教育記録、巡視記録、KY活動記録、機械点検表 | 労安法、共通仕様書 | 全期間 |
| **材料承認** | 使用材料承認願、品質証明書、ミルシート、カタログ | 共通仕様書（材料編） | 材料搬入前 |
| **電子納品** | SXF図面、工事写真JPEG、XML管理ファイル | 電子納品要領、CAD製図基準 | 完了時 |
| **段階確認** | 段階確認書、確認状況写真 | 共通仕様書（段階確認一覧） | 施工の節目 |
| **検査書類** | 上記全て + 施工体制台帳、打合せ簿、リサイクル書類等 | 工事成績評定要領 | 中間/完成検査 |

### 1-2. 業務委託（設計・測量・調査）

| カテゴリ | 主な書類 | チェック基準 |
|---------|---------|-------------|
| **照査報告書** | 設計条件の確認、計算の照査、図面の照査、数量の照査 | 設計業務等共通仕様書 |
| **設計計算書** | 適用基準、設計条件、計算過程、使用ソフト | 道路橋示方書、道路構造令等 |
| **数量計算書** | 図面寸法との整合、計算過程、単位・桁数 | 土木工事数量算出要領 |
| **図面** | レイヤ名、線色・線種、文字サイズ、ファイル形式 | CAD製図基準 |

### 1-3. 包括委託特有の確認書類

包括委託では上記に加えて、以下の書類が日常的に発生する:

| 書類 | 内容 | 頻度 |
|------|------|------|
| **パトロール報告書** | 道路パトロール結果、損傷箇所の記録 | 日次〜週次 |
| **補修実施報告書** | 補修内容、使用材料、前後写真 | 補修の都度 |
| **年次報告書** | 受託区間全体の維持管理実績、損傷傾向分析 | 年次 |
| **緊急対応報告書** | 災害時・事故時の対応記録 | 発生時 |
| **住民対応記録** | 苦情・要望への対応記録 | 随時 |

## 2. 自治体職員の現状と課題

### 深刻な技術職員不足

- 約9割の「村」、約5割の「町」で土木部門職員が**10人未満**
- 1人の監督員が**複数工事を同時に担当**するのが常態化
- 技術系職員の採用試験で**受験者ゼロ**のケースも発生

### 書類チェックの負担

```
                          ┌─ 施工計画書チェック
                          ├─ 出来形管理値の確認
       ┌─ 工事A ─────────┼─ 品質管理書類の確認
       │                  ├─ 工事写真の確認
       │                  └─ 材料承認の処理
       │
監督員 ─┼─ 工事B ─────────── (同上)
       │
       ├─ 工事C ─────────── (同上)
       │
       └─ 窓口対応、予算管理、住民対応...
```

### 具体的なリスク

| リスク | 発生原因 | 影響 |
|--------|---------|------|
| 出来形管理値の確認不足 | 大量の測定データを目視チェック | 規格値超過の見落とし → 手戻り工事 |
| 材料承認の形骸化 | 精査する時間がない | 不適切な材料の使用を見逃す |
| 段階確認の省略 | 人手不足で立会いが間に合わない | 不可視部分の品質未確認 |
| 設計図書の照査不足 | 基準書の横断的確認に時間がかかる | 設計と施工の齟齬を現場で発見 |
| 基準改定の見落とし | 毎年改定される仕様書の追従 | 旧基準での確認 |

## 3. 自動チェックの現状と空白地帯

### 既に自動化されている領域

| 領域 | ツール | 自動化レベル |
|------|--------|-------------|
| 電子納品（形式チェック） | 国交省チェックシステム（無償） | ✅ 実用化済み |
| CAD製図基準（形式チェック） | EX-TREND武蔵、SXF Viewer | ✅ 実用化済み |
| 施工計画書（作成支援） | EARTHBRAIN、NITACO、大成建設 | ✅ AI作成支援は実用化 |
| 工事写真（分類・台帳） | 蔵衛門AI、Cheez | ✅ 実用化済み |

### 自動化が進みつつある領域

| 領域 | ツール | 自動化レベル |
|------|--------|-------------|
| 設計図書の整合性チェック | 東芝CISSART | 🔶 北陸地整で採用 |
| 図面⇔計算書の整合性 | 建設技術研究所 | 🔶 社内実用化、社外提供検討中 |
| 積算ミス検知 | 富士通ESTIMA | 🔶 長野県で実証完了 |
| 配筋検査 | 清水建設等 | 🔶 現場実証段階 |

### **空白地帯（=提案チャンス）**

| 領域 | 現状 | 自治体の痛み |
|------|------|-------------|
| **発注者の書類チェック支援** | 受注者向けツールは豊富だが、発注者向けはほぼゼロ | 最も深刻 |
| **書類内容の基準適合チェック** | 形式チェックは自動化済み、内容チェックは人手 | チェック漏れリスク |
| **共通仕様書との自動照合** | 人が仕様書を読んで確認 | 基準改定時に特に困る |
| **複数工事の横断管理** | 工事ごとに別管理 | 全体像が見えない |
| **維持管理段階の書類管理** | 施工完了後、書類がバラバラに | 包括委託で特に問題 |

## 4. 提案: CiviLinkを基盤とした書類チェックシステム

### コンセプト

```
受注者（施工会社）                    発注者（自治体）
                                    
 図面・書類をCiviLinkにアップロード     
        ↓                           
 ┌─────────────────────────┐         
 │   CiviLink + チェック機能  │         
 │                           │         
 │  ① 形式チェック（自動）    │ → エラーリスト自動生成
 │  ② 基準照合（AI支援）     │ → チェック結果ダッシュボード
 │  ③ 指摘管理（既存機能）    │ → 是正指示・回答・完了
 │  ④ 履歴蓄積（既存機能）    │ → 監査証跡・ナレッジ
 │                           │         
 └─────────────────────────┘         
        ↓                           
 電子納品成果品として出力              
```

### フェーズ別実現ロードマップ

#### Phase 1: CiviLink現行機能で今すぐできること（追加開発なし）

| 対象書類 | CiviLinkの活用方法 | 自治体のメリット |
|---------|-------------------|----------------|
| **図面の照査** | 図面にマークアップ・コメントで照査結果を記録 | 赤黄チェックをデジタル化、履歴が残る |
| **図面のバージョン管理** | 設計変更のたびにバージョンを管理、差分を可視化 | 「どの版が最新か」の混乱解消 |
| **段階確認の記録** | 確認箇所を図面上にマーキング + 写真添付 | 確認記録が図面と紐付いて蓄積 |
| **複数工事の図面管理** | プロジェクト別にフォルダ管理、検索 | 担当工事の図面を一元管理 |
| **補修記録の管理** | 補修前後の図面・写真をバージョンとして管理 | 包括委託の補修履歴が図面に蓄積 |

**→ これだけでも「自治体監督員のための図面チェック基盤」として提案可能**

#### Phase 2: 軽量な自動チェック機能（小規模開発）

| 機能 | 概要 | 技術難易度 |
|------|------|-----------|
| **チェックリストテンプレート** | 共通仕様書の確認項目をテンプレート化。図面を開くと該当工種のチェック項目が自動表示される | ★☆☆ |
| **出来形管理値の規格値チェック** | 測定値を入力 → 規格値（共通仕様書の工種別基準値）と自動照合 → OK/NG判定 | ★★☆ |
| **電子納品フォルダ構成チェック** | CiviLinkにアップされた図面が電子納品要領の命名規則・構成に準拠しているか自動検証 | ★☆☆ |
| **図面差分レポート自動生成** | バージョン間の差分を自動検出し、「変更箇所一覧」レポートを出力 | ★★☆ |

#### Phase 3: AI活用による高度な自動チェック（中長期開発）

| 機能 | 概要 | 参考事例 | 技術難易度 |
|------|------|---------|-----------|
| **図面⇔数量計算書の整合性チェック** | AI+OCRで図面から工種・規格・数量を読み取り、数量計算書と自動対比。不整合箇所を指摘として自動生成 | 東芝CISSART | ★★★ |
| **共通仕様書RAGアシスタント** | 共通仕様書・設計基準をベクトルDBに格納。「この設計は基準を満たしているか？」に根拠条文付きで回答 | （実用化例なし → 先行者利益） | ★★★ |
| **施工計画書の自動照査** | 提出された施工計画書の必須記載事項の網羅性チェック、特記仕様書との整合性確認 | NITACO施工計画書AI | ★★★ |
| **積算ミス検知** | 過去の積算データとの比較で異常値を自動検出 | 富士通ESTIMA（長野県実証） | ★★★ |
| **写真管理基準チェック** | 工事写真の撮影項目・頻度が管理基準を満たしているか自動検証 | 蔵衛門AI | ★★☆ |

## 5. 包括委託文脈での提案ストーリー

### 佐賀市への提案として

```
【Before】現状の包括委託
  施工会社: 紙の報告書、バラバラの写真、属人的な図面管理
  自治体: 大量の紙書類をチェック、基準との照合は人手頼み

       ↓ CiviLink導入 ↓

【After】デジタル化された包括委託
  施工会社: CiviLinkで図面・報告書を一元管理・提出
  自治体: CiviLink上でチェック、基準照合はシステムが支援

       ↓ 将来的にAI機能追加 ↓

【Future】AI支援付き包括委託
  施工会社: AIが書類ドラフトを作成、CiviLinkで提出
  自治体: AIがチェック支援、職員は判断に集中
```

### 他の包括委託DXとの棲み分け

| 領域 | 既存ツール | CiviLink |
|------|-----------|----------|
| 台帳管理・資産管理 | Genavis Tao-Asset | - |
| AI舗装診断 | Draw-AI | - |
| 通報管理・住民対応 | コールセンターシステム | - |
| **図面管理・照査** | **（なし）** | **CiviLink** |
| **書類の基準適合チェック** | **（なし）** | **CiviLink + チェック機能** |

→ **図面起点の書類チェック**は包括委託DXの空白地帯。CiviLinkの独自ポジション。

## 7. 実現に向けた論点

### すぐ進められること

- [ ] CiviLink現行機能で「自治体監督員向けの使い方ガイド」を作成
- [ ] 共通仕様書の確認項目一覧をテンプレート化（Excelまたはmd形式でまず整理）
- [ ] 佐賀市の共通仕様書（独自版）を入手し、国交省版との差異を確認

### 開発判断が必要なこと

- [ ] Phase 2の軽量チェック機能は開発コスト vs 効果を検討
- [ ] Phase 3のAIチェック機能は技術検証（PoC）が必要
- [ ] 八千代エンジニヤリングのオープンソース照査ソフトの調査（流用可能性）

### パートナー連携の可能性

- [ ] 東芝CISSART: API連携でチェック結果をCiviLinkに取り込み
- [ ] 富士通ESTIMA: 積算チェック結果との連携

## 8. 4/20佐賀市訪問での使い方

この提案をそのまま資料として使うのではなく、**ヒアリングの武器**として活用する:

### 聞くべきこと

1. 「書類チェックで一番時間がかかっているのは何ですか？」
2. 「出来形管理の規格値チェックは手作業ですか？」
3. 「共通仕様書は佐賀市独自版がありますか？国交省版ベースですか？」
4. 「電子納品はどこまで義務化されていますか？」
5. 「監督員1人あたり何件の工事を担当していますか？」

### 見せ方

- Phase 1（CiviLink現行機能）をデモで見せる
- Phase 2-3は「将来こういうこともできます」程度に触れる
- **「自治体職員のチェック業務を楽にする」**というメッセージが最重要
- 技術的な話より、「何時間節約できるか」の方が刺さる
