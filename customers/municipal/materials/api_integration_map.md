## 詳細

### ◎ API連携が可能（自動化・リアルタイム連携が実現）

#### 1. LINE WORKS
- **API:** REST API 2.0（フル機能）
- **認証:** OAuth 2.0（JWT方式のService Account対応）
- **主要API:** Bot（メッセージ送受信）、Calendar、Drive、Contact、Audit
- **Webhook:** Bot callback URLでメッセージ受信をトリガー可能
- **CiviLink連携案:**
  - CiviLink上の承認完了 → LINE WORKS Botで関係者に自動通知
  - LINE WORKSでの現場報告 → CiviLinkの図面コメントに自動転記
  - 段階確認の日程調整 → Calendar APIで自動調整
- **開発者ポータル:** https://developers.worksmobile.com/jp/docs/api
- **パートナープログラム:** あり（App Directory公開可能）

#### 2. クラウドサイン (CloudSign)
- **API:** REST API（SwaggerHub公開済み）
- **認証:** APIキー方式
- **主要API:** 書類作成（POST /documents）、参加者設定、PDF取得、書類一覧
- **Webhook:** 契約完了・却下・バウンスの通知（最大20 URL設定可）
- **CiviLink連携案:**
  - CiviLink上の承認WF完了 → クラウドサインで自動契約書作成
  - 契約完了Webhook → CiviLinkの案件ステータスを自動更新
  - 契約PDFをCiviLinkの図面フォルダに自動格納
- **API Doc:** https://app.swaggerhub.com/apis/CloudSign/cloudsign-web_api/
- **連携済み外部:** Salesforce, kintone, Box, Slack（100+サービス）

---

### ○ プロトコルレベルで連携可能

#### 3. GNSS/RTK（ジェノバVRS / docomo IoT GNSS）
- **プロトコル:** Ntrip（Networked Transport of RTCM via Internet Protocol）
- **データ形式:** RTCM3.x（業界標準）
- **接続:** ntrip.jenoba.jp / d-gnss.jp（ID/Pass認証）
- **CiviLink連携案:**
  - 測量データ（座標）をCiviLinkの図面上にプロットする際の基盤
  - 直接API連携というよりは、測量機器・ソフトから出力された座標データをCiviLinkが受け取る形
- **備考:** 全てのNtrip互換GNSS受信機で利用可能。CiviLink側で座標取り込み機能があれば連携成立

---

### △ ファイルベースで連携可能（自動化には要カスタム開発）

#### 4. 蔵衛門 (Kuraemon)
- **API:** なし
- **出力形式:** 電子納品XML（国交省準拠）、PDF、SXF（P21）CADデータ
- **CiviLink連携案:**
  - 蔵衛門のXML出力 → CiviLinkの電子納品チェック機能に取り込み
  - 工事写真PDFをCiviLinkの図面フォルダにインポート
  - **課題:** 手動エクスポート→インポートが必要。自動連携にはRPA的な仕組みが必要
- **将来可能性:** 蔵衛門側にAPI要望を出すか、ファイル監視で自動取り込みする仕組みを構築

#### 5. ひびみっけ (Hibimikke / Fujifilm)
- **API:** なし
- **入力:** JPEG画像（一眼レフ・コンパクトカメラ・ドローン撮影）
- **出力:** JPEG合成画像、CADデータ（ひび割れ形状）、Excelスプレッドシート（幅・長さ計測値）
- **処理:** ローカルソフトでクラウドにアップロード → 約60分後にダウンロード
- **CiviLink連携案:**
  - CAD出力（ひび割れ形状データ）→ CiviLinkの図面上にオーバーレイ表示
  - Excel計測値 → CiviLinkの橋梁データベースに取り込み
  - **課題:** バッチ処理型のため、ファイルの手動受け渡しが基本
- **将来可能性:** Fujifilmに直接API連携を打診する価値あり（NETIS登録技術のため需要は高い）

#### 6. 積算AI (KK Generation)
- **API:** なし
- **入力:** PDF、CAD、BIM図面
- **出力:** Excelテンプレート（数量計算書・見積書）、DXF
- **CiviLink連携案:**
  - CiviLink上のPDF図面 → 積算AI にアップロード → Excel結果をCiviLinkの数量整合チェックに入力
  - **課題:** SaaS型のため手動操作が前提。スタートアップ段階でAPI開発の余地あり
- **将来可能性:** 創業2023年のスタートアップ。早期のパートナーシップ打診でAPI連携を共同開発する余地大

#### 7. 工程表AI（NITACO / Tsukunobi）
- **API:** なし
- **入力:** 設計図書・過去書類
- **出力:** 施工計画書初稿、DXF（CAD Trace AI）
- **CiviLink連携案:**
  - CiviLink上の設計図書をNITACOにエクスポート → 工程表を取り込み
  - **課題:** ファイルベースの受け渡しが前提
- **将来可能性:** BPOサービス中心のため、API開発への投資優先度は低い可能性

#### 8. Dr.Bridge (日本海コンサル × BIPROGY)
- **API:** なし
- **入力:** 点検写真＋橋梁諸元データ
- **出力:** AI診断結果（健全度・劣化要因）、点検調書の自動生成
- **CiviLink連携案:**
  - CiviLink上の点検写真・橋梁データ → Dr.Bridge に入力 → 診断結果をCiviLinkに反映
  - **課題:** BIPROGYのエンタープライズチャネルを経由する必要あり
- **将来可能性:** BIPROGY（旧日本ユニシス）はAPI公開に慣れた企業。交渉余地あり

---

### × 連携困難（マネージドサービス/クローズド）

#### 9. みちログ (Aisin)
- **API:** なし。データ出力形式も非公開
- **提供形態:** 自治体向けマネージドサービス（車載カメラ＋エッジAI＋クラウド）
- **CiviLink連携案:** 現状では困難。みちログのデータが自治体のオープンデータとして公開される場合にのみ利用可能
- **将来可能性:** Aisinに直接アプローチ。ただし自動車部品メーカーのIoT事業部門のため、建設DXベンダーとの連携優先度は未知数

#### 10. Cheez（写真台帳AI）
- **API:** なし。出力形式も非公開（アプリ完結型）
- **提供形態:** スマホアプリ（写真撮影→AI→台帳自動生成）
- **CiviLink連携案:** 現状では困難。台帳PDFの手動アップロードが限界
- **将来可能性:** Open Network Lab採択のスタートアップ。早期のパートナーシップ打診で連携機能の共同開発を提案する余地あり

## 提案への反映方針

### 4/20佐賀市訪問での見せ方
- LINE WORKS・クラウドサインは「すぐにCiviLinkと自動連携できる」と伝える
- 蔵衛門・ひびみっけは「ファイルベースで連携、将来的にはAPI化を推進」
- 全体として「CiviLinkがハブになり、各ツールのデータを図面に集約する」ストーリー

### 技術的示唆
- CiviLink側に必要な機能: **汎用ファイルインポートAPI**（Excel, CAD, XML, PDF）
- これがあれば、API未対応ツールもファイル経由で半自動連携が可能になる
- LINE WORKS Bot連携はPOC Phase 1で最初に実装すると体験価値が高い
