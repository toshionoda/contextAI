# 北九州市 インフラ管理の課題レポート

**作成日:** 2026-04-04
**目的:** 自治体向けCiviLink展開に向けた北九州市のインフラ管理課題の整理

---

## 1. 北九州市の概況

- **人口:** 約90万人（2025年9月時点で90万494人）、90万人割れが目前
- **特徴:** 100年以上の都市インフラ整備の歴史を持つ政令指定都市
- **財政:** 福祉・医療費が年々増加し、施設整備の投資的経費はピーク時から大幅減少・横ばい

## 2. インフラ老朽化の現状

### 2.1 施設の高齢化
- 昭和40〜50年代（1965〜1980年代）に集中整備された公共施設が、今後10数年で**一斉に更新時期**を迎える
- 一部施設は建設後の経過年数が**耐用年数に到達**しており、今後さらに増加見込み
- 国全体では2033年までに道路橋63%、トンネル42%、下水道管25%が建設後50年超

### 2.2 北九州市の主要インフラ
| 分野 | 状況 |
|------|------|
| **橋梁** | 長寿命化修繕計画を策定済（若戸大橋編：R5年3月策定） |
| **道路** | 予防保全型メンテナンスサイクルへの転換を推進 |
| **上下水道** | 老朽化による布設替え工事が進行中 |
| **港湾** | 社会インフラ版マネジメント基本計画の対象 |

## 3. 主要課題

### 3.1 人口減少と財政制約
- 人口90万人割れが目前 → 税収減少 → インフラ維持管理費の確保が困難に
- 福祉・医療費増 vs 投資的経費減のトレードオフが深刻化
- **更新費用の集中**：10数年後に一斉更新が発生するリスク

### 3.2 技術職員の不足
- 全国の市町村で技術系職員がいない割合は約3割（国交省データ）
- 北九州市も例外ではなく、点検・診断・措置・記録のメンテナンスサイクルを回す人材確保が課題
- ベテラン技術者の退職に伴う**技術・ノウハウの継承**問題

### 3.3 膨大な施設数と多様性
- 100年以上にわたり整備された多種多様な社会インフラを保有
- 建設年代・工法・材料がバラバラで、統一的な管理が困難
- 施設台帳の電子化・一元管理の遅れ

## 4. 北九州市のDX・i-Construction推進状況

### 4.1 組織体制
- **DX・AI戦略室**を設置し、市全体のDXを推進
- **北九州市i-Construction推進協議会**を設置（官民連携）

### 4.2 主な施策
- **公共工事関係部署のDX推進プラン**（R4年4月策定）
- ICT活用工事の試行（受注者希望型：土工・舗装工）
- 公共施設マネジメント実行計画（H28策定、R4一部見直し）
- 社会インフラ版マネジメント基本計画

### 4.3 DX推進の課題
- ICT活用工事はまだ**受注者希望型の試行段階**（一部工種のみ）
- 建設プロセス全体（調査→設計→施工→検査→維持管理）の一気通貫デジタル化は道半ば
- 中小建設業者のICTリテラシー・導入コストの壁

## 5. CiviLinkの提案機会

### 5.1 刺さりそうなポイント
| 課題 | CiviLinkの価値提案 |
|------|---------------------|
| 技術職員不足 | 図面・文書の一元管理で少人数でも効率的に管理可能 |
| 施設台帳の分散 | クラウドベースの統合管理プラットフォーム |
| 点検記録の標準化 | 現場での図面閲覧・マーキング・記録のデジタル化 |
| 技術継承 | 過去の点検履歴・対応記録のナレッジ蓄積 |
| DX推進プランとの整合 | 市のDX方針に合致するソリューションとして提案可能 |

### 5.2 アプローチ案
1. **北九州市i-Construction推進協議会**へのコンタクト（官民連携の枠組みを活用）
2. 大学同期経由で**建設局or上下水道局**の担当者紹介を依頼
3. まずは**橋梁点検・道路パトロール**の図面閲覧ニーズからアプローチ
4. DocuWorks→PDF移行の知見を活かし、紙図面のデジタル化支援も合わせて提案

### 5.3 注意点
- 自治体は**調達プロセス**（入札・随意契約等）があるため、民間とはリードタイムが異なる
- 実証実験やPoC枠での導入が現実的な最初のステップ
- セキュリティ・個人情報保護に対する要件が民間より厳格
