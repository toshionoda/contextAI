## 4. 道路設計技術者との意見交換 進行台本（メイン）

### 診断：なぜ刺さらなかったか

道路設計の図面は**枚数が少ない**けれど、**1枚あたりの密度が異常に高い**仕事。CiviLinkを「**たくさんの図面を効率的に照査するツール**」として説明すると道路には全く響かない。

道路設計者の真のペインは「枚数」ではなく：

| 真のペイン | 何が起きているか |
|---|---|
| **協議の往復回数** | 1枚の平面図に発注者・警察・自治会・地権者から赤書き指摘が何十回 |
| **断面間の整合** | 平面・縦断・横断・構造物詳細が4方向で連動。1箇所直すと全部直す |
| **基準・要領との適合** | 道路構造令、設計要領、地方整備局基準。観点が膨大 |
| **A案/B案/C案の比較** | それぞれの図面と判断理由がバラバラ |
| **過去案件の参照** | 同じ路線の上下流で「あのとき何故その判断をしたか」が再現できない |
| **外注成果品の照査** | 自分で書かず、コンサル・協力会社からの納品物を指摘して返す仕事 |

→ **「外注成果品の照査」と「協議指摘の管理」はCiviLinkど真ん中**。これを引き出すのが今日のゴール。

### 90分の進行設計

| Phase | 時間 | やること | 狙い |
|---|---|---|---|
| 1. アイスブレイク | 10分 | 「最近どんな案件を？」「どこの整備局担当？」 | 信頼構築。栗山さんに紹介役を頼む |
| 2. **業務観察ヒアリング** | 30分 | 「直近1ヶ月で時間がかかった業務TOP3」を全員に聞く | 真のペインを引き出す。CiviLinkの話はまだしない |
| 3. ペインの再フレーミング | 15分 | TOP3に対し「もし○○ができたら？」と仮説を投げる | CiviLinkに繋がる接点を発見 |
| 4. **ライブ共創ワーク** | 25分 | 「貴社の現案件の図面1枚、Webで開いて触ってみる」 | 触感で気づきを生む |
| 5. クロージング | 10分 | 「明日からの第一歩はこれにしませんか」 | 具体的な次アクション合意 |

### 冒頭の宣言（売り手モードを脱する）

> **「貴社の道路設計の現場で、CiviLinkがどうあれば本当に役に立つのか、私達は分かっていません。今日はそれを教えていただきに来ました。今のCiviLinkでは刺さらないかもしれません。だからこそ、何があれば刺さるのかを教えていただきたいんです」**

これを冒頭で言い切る。売り手⇒買い手 を脱して **共同設計者** に立ち位置を変える。

### Phase 2 で必ず聞く7つの質問（順番が大事・CiviLinkの話はしない）

1. 直近やっている道路案件の**規模・段階**（予備設計／詳細設計／実施設計）は？
2. 1案件で扱う図面の**種類と枚数**はおよそ？（平面・縦断・横断・構造物詳細）
3. **外注の使い方**は？（自社で全部書く／一部外注／フル外注）
4. 発注者からの**赤書き指摘、どう管理**してますか？前回の指摘がどう反映されたか後で確認するときは？
5. **外注先から上がった成果品を照査するとき、何を見ますか**？どこに記録しますか？
6. 同じ路線の上下流の**過去案件で何故その判断をしたかを確認**したいとき、どうしてますか？
7. **若手が独り立ちするまでの育成ボトルネック**は何だと思いますか？

→ ほぼ確実に、**4・5・6**で「うわっ、それ困ってる」が出る。そこを起点にPhase 3へ。

### Phase 3 の仮説ぶつけ（CiviLinkの繋ぎ方）

**質問4が刺さったら**：「**発注者協議の赤書きを、CiviLinkで全部一元管理して『どの指摘がどう反映されたか』が後から検索できたら、どうですか**」

**質問5が刺さったら**：「**外注先の成果品をCiviLink上で照査して、指摘がそのまま外注先に返って、修正後の差分が自動で見られたら、どうですか**」

**質問6が刺さったら**：「**過去の同じ路線で出た指摘を、座標で検索できたら、どうですか**」（指摘1000件溜まっている事実を伝える）

**質問7が刺さったら**：「**ベテランの指摘の付け方を若手が後追いで学べるナレッジになります**」（属人化解消）

### Phase 4 ライブワーク準備

- 栗山さんか渋谷さんに**事前に頼んで**、道路案件の図面1枚をCiviLinkに上げて触る
- 机上の説明ではなく**触感**で「あ、これ使える」を作る
- 「**この場でアカウント発行します、明日から触れますよ**」を準備

## 8. 想定Q&A

### Q1: 「照査様式自動連携、結局いつ実装されるんですか？」
**A**: 「ご要望をいただいてから2ヶ月、ロードマップ整理に時間をいただき大変申し訳ございませんでした。複数顧客から同様要望があり、開発優先度は高い。Year 2（2027年）での実装を視野にいれて中期計画に組み込んでいます。先行ユーザーとしてのフィードバックを設計に反映させていただきたいので、要件定義のセッションを5月に1回設けさせていただけませんか」

### Q2: 「印刷バグはもう直ってますか？」
**A**: 当日の直近修正状況を**事前確認**して回答準備。直っていれば「修正済みでお手元で確認可能です」、未対応箇所があれば誠実に開示＋対応スケジュール提示。

### Q3: 「うちの道路設計、図面少ないからCiviLink要らないんじゃない？」
**A**: 「枚数の話ではなく、**1枚に対する協議の往復・外注成果品の照査・過去案件の参照**が真の効果領域だと考えています。今日のヒアリングで、貴社の道路設計のどこにCiviLinkが効くかを一緒に見つけさせてください。もし本当に要らないなら、それも貴重なフィードバックです」

### Q4: 「他社の道路設計で導入実績ありますか？」
**A**: 「橋梁・水工・建築では実績あります。**道路設計部門での本格導入は、貴社が先行ユーザーとなる可能性大**です。だからこそ要件を一緒に作りたい。先行ユーザーには中期ロードマップへの優先反映というメリットも提示できます」

### Q5（社長想定）: 「価格は？年間いくら？」
**A**: 「現契約は5名×6ヶ月で30万円。仮に道路設計部門20名×12ヶ月で240万円規模。ただしROI観点では、人時生産性向上で年間数千万円規模のリターンを想定しています。ご導入のされ方次第で柔軟にプラン設計可能です」

### Q6（社長想定）: 「マルメさんの会社規模は？継続性は？」
**A**: 「マルメは設立XX年、CiviLinkは2024年に本格商用化、有償化16社。投資家からも複数の関心をいただいており（JAFCO等）、事業継続性に懸念はありません。**長期パートナーシップを前提**にお付き合いさせていただきたい」
