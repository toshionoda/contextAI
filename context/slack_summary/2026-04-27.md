## 1. CiviLink開発：DocuWorksファイル変換中UX改善

**チャンネル:** #dev-civilink  
**主な参加者:** 高田 真矢、橋田 和樹、中山 幸葉

**背景・問題提起（高田 真矢）**
- DocuWorksファイルがPDF変換中の場合、**ファイルを開いた後**に進捗ダイアログが表示される仕様になっている（添付1の挙動）
- Autodesk系ファイルは**開く前**に変換処理中ダイアログが表示される（添付2の挙動）
- DocuWorks系も開く前に知らせる形に統一したいと相談

**議論の流れ**

| 発言者 | 内容 |
|--------|------|
| 橋田 和樹 | DocuWorksに統一して欲しい |
| 中山 幸葉 | 開く前ダイアログは冗長になるため橋田さん案でDocuWorks系に統一。Issue作成してSoraチームにアサインする |
| 高田 真矢 | ファイル管理ページのテーブルにプログレスバー＋「処理中のため閲覧不可」ツールチップも追加したいが過剰か？ |
| 橋田 和樹 | ファイル管理画面の読み込み負荷の問題あり。双方向同期になるのか確認 |
| 中山 幸葉 | 現在は進捗ポーリングで都度APIアクセス発生中。WebSocket通知に変更するか、ファイル管理画面にスピナー＋ツールチップを出すことは技術的には可能。ただし別Issueで検討が妥当 |
| 橋田 和樹 | WebSocket開始手続きも複雑になりそう。ファイル管理画面からの要望が上がってからでよい |
| 高田 真矢 | 合意。APIコスト・パフォーマンスを考えると簡単でないので声が上がってから検討 |

**決定事項**
- 変換中ダイアログはDocuWorks系に統一（Sora チームへIssueアサイン）
- ファイル管理ページのプログレスバー表示は別Issueで検討、ユーザー要望が上がってから判断

**未解決事項**
- WebSocket化 or ポーリング継続の技術選定（別Issue）
