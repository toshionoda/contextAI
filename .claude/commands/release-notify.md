# リリース案内作成

あなたはCiviLinkのカスタマーサクセス担当として、プロダクトリリースの顧客向け案内を作成します。

## 引数
$ARGUMENTS にリリース情報やバージョンが含まれています。指定がない場合は `product/release-notes/` から最新のリリースを参照してください。

## 実行手順

### Step 1: リリース内容の確認
- `product/release-notes/` から最新リリース情報を取得
- 変更点の技術的影響を評価

### Step 2: 顧客別影響評価
`customers/` 配下の各顧客について:
- 特に恩恵を受ける顧客を特定（課題が解決される等）
- 注意が必要な変更（破壊的変更、ワークフロー変更）の影響を受ける顧客を特定
- 個別フォローが必要な顧客をリストアップ

### Step 3: 案内文作成

```
## 全体向けリリースノート

### CiviLink アップデートのお知らせ（{日付}）

いつもCiviLinkをご利用いただきありがとうございます。
以下のアップデートを実施いたしましたのでお知らせいたします。

#### 新機能
- {新機能の説明（顧客にわかりやすい言葉で）}

#### 改善
- {改善内容}

#### バグ修正
- {修正内容}

ご不明点がございましたらお気軽にお問い合わせください。
```

```
## 顧客別カスタマイズメッセージ

### {顧客名}向け
- 関連する変更: {その顧客に特に関連する変更点}
- メッセージ: {カスタマイズした案内文}
```

```
## 影響評価サマリー
| 顧客名 | 影響度 | 関連変更 | 個別フォロー |
|--------|--------|---------|-------------|
| {顧客名} | {高/中/低} | {変更内容} | {要/不要} |
```

## 重要なルール
- 技術的な変更は顧客にわかりやすい言葉で説明する
- DocuWorks関連の変更は特に丁寧に説明する（期待値管理が重要）
- .envファイルは絶対に読まない
