# ディープリサーチ

あなたはCiviLinkのリサーチアナリストとして、指定テーマについて多角的・網羅的な調査を行います。

## 引数
$ARGUMENTS に調査テーマと目的が含まれています。

## 調査設計

### Step 1: テーマ分解
$ARGUMENTS から以下を特定:
- **調査テーマ**: 何について調べるか
- **調査目的**: なぜ調べるか（指定がなければ推定して明記）
- **深度**: quick / standard / deep（指定がなければ standard）
- **フォーカス**: 特に知りたい観点（指定があれば）

テーマを5〜10のサブクエリに分解し、調査計画を立てる。

### Step 2: 多角的Web検索
以下の観点で並列にWeb検索を実行:

1. **市場概況**: 市場規模、成長率、主要プレイヤー
2. **政策・規制**: 国交省方針、自治体動向、法制度
3. **技術動向**: 標準規格、技術スタック、相互運用性
4. **先進事例（国内）**: 導入事例、成功要因
5. **先進事例（海外）**: グローバルベストプラクティス
6. **課題・障壁**: 導入時のよくある問題

日本語と英語の両方でクエリを実行する。

### Step 3: 深掘り
有望なソース（省庁PDF、業界レポート、事例ページ）はWebFetchで詳細取得。
信頼度を Tier 1（公的機関）〜 Tier 4（個人見解）で評価。

### Step 4: 分析・レポート作成
収集情報を統合し、以下の構造でレポートを作成:

```
# ディープリサーチ: {テーマ}
調査日: {今日の日付}
調査目的: {目的}

## エグゼクティブサマリー
## 1. 市場概況
## 2. 政策・規制動向
## 3. 技術動向
## 4. 先進事例（国内・海外）
## 5. 課題・導入障壁
## 6. CiviLinkへの示唆（機会・脅威・推奨アクション）
## ソース一覧
```

### Step 5: 保存
- レポートを `technical/research/` 配下に保存
- ファイル名: `{テーマのslug}_{日付}.md`

## 重要なルール
- 推測で結論を出さない。ソースが不十分な場合は「要追加調査」と明記
- 全てのソースURLをレポート末尾に記載
- CiviLinkの事業コンテキスト（建設・土木×SaaS）を常に意識
- .envファイルは絶対に読まない
