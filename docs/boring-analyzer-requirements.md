## 1. プロジェクト概要

### 1.1 目的

土木設計者が、国交省電子納品XML（BED形式）のボーリングデータをアップロードするだけで、設計に必要な地盤情報を自動抽出・算出できるWebアプリケーションを開発する。

### 1.2 背景

- ボーリングデータから設計条件を整理する作業は、設計者が手作業で行っているケースが多い
- N値の集計、支持層判定、地盤定数推定などの定型作業に時間がかかる
- 電子納品XMLは構造化データであるにもかかわらず、活用ツールが限られている

### 1.3 対象ユーザー

| ユーザー種別 | 想定利用シーン |
|-------------|--------------|
| 土木設計技術者 | 基礎設計・地盤設計の事前検討 |
| 地質調査技術者 | 調査結果の品質確認・レポート作成 |
| 建設コンサルタント | 概略設計段階の地盤条件整理 |

### 1.4 公開方針

- 初回リリースからインターネット上に一般公開する
- 無料で利用可能とする（将来の有償化は検討事項として保留）

## 2. システム構成

### 2.1 アーキテクチャ

```
[ユーザー（ブラウザ）]
    ↓ XMLファイルアップロード
[Vercel CDN]
    ↓
[Next.js App Router]
    ├── クライアントサイド: XMLパース、結果表示、グラフ描画
    └── サーバーサイド (Serverless Functions): PDF/Excel生成
    ↓
[ユーザー（ブラウザ）]
    ← 解析結果表示 / ファイルダウンロード
```

### 2.2 設計方針

| 方針 | 内容 | 理由 |
|------|------|------|
| ステートレス構成 | アップロード→解析→出力を1リクエストで完結 | DB不要、運用コスト最小化 |
| データ非保存 | ユーザーのXMLデータはメモリ上でのみ処理 | プライバシー保護、サーバー管理不要 |
| クライアントサイドパース | XMLパースはブラウザ側で実行 | サーバーレス関数の実行時間・メモリ制約を回避 |
| TypeScript統一 | フロントエンド・バックエンドともTypeScript | 個人開発での認知負荷最小化 |

### 2.3 技術スタック

| レイヤー | 技術 | 用途 |
|---------|------|------|
| フレームワーク | Next.js (App Router) + TypeScript | フルスタック |
| スタイリング | Tailwind CSS | レスポンシブUI |
| グラフ描画 | Recharts | N値分布グラフ等 |
| 図面描画 | react-konva (Canvas API) | 柱状図・断面図（Phase 3） |
| XMLパース | fast-xml-parser | BED形式XMLの解析 |
| Excel生成 | exceljs | xlsx出力 |
| PDF生成 | pdfmake | PDF出力 |
| ホスティング | Vercel (Hobby Plan) | 無料枠で運用 |

### 2.4 インフラ要件

| 項目 | 仕様 |
|------|------|
| ホスティング | Vercel Hobby Plan（無料） |
| CDN | Vercel Edge Network（グローバル） |
| サーバーレス関数 | メモリ1024MB、実行時間10秒 |
| 月間帯域 | 100GB（無料枠） |
| ドメイン | 初期: xxx.vercel.app / 将来: 独自ドメイン |
| DB | なし（ステートレス構成） |
| 月額コスト | 0円 |

## 3. 入力仕様

### 3.1 対応フォーマット

| フォーマット | 対応 | 備考 |
|-------------|------|------|
| 国交省 電子納品XML（BED形式） | 必須 | ボーリング交換用データ |
| その他 | 対象外 | Phase 1ではBEDのみ |

### 3.2 BED形式の対象要素

BED（Borehole Exchange Data）形式は、国交省「地質・土質調査成果電子納品要領」に基づくXML形式。以下の要素をパース対象とする。

| XML要素 | 抽出内容 | 必須/任意 |
|---------|---------|----------|
| ボーリング基本情報 | ボーリング名、座標、孔口標高、掘進長、調査日 | 必須 |
| 地層区分 | 層番号、上端深度、下端深度、土質名、地質年代、色調 | 必須 |
| 標準貫入試験 | 試験深度、N値、貫入量、打撃回数 | 必須 |
| 地下水位 | 測定日、水位深度 | 任意 |
| 室内試験結果 | 粒度試験、一軸圧縮試験等 | 任意（Phase 2で活用） |
| プロジェクト情報 | 事業名、調査会社、調査者 | 任意 |

### 3.3 アップロード仕様

| 項目 | 仕様 |
|------|------|
| アップロード方法 | ドラッグ&ドロップ / ファイル選択ダイアログ |
| 複数ファイル | 対応（一度に複数のBED XMLをアップロード可能） |
| ファイルサイズ上限 | 10MB/ファイル |
| ファイル数上限 | 50ファイル/1回 |
| バリデーション | XML構文チェック、BEDスキーマ適合チェック |
| エラーハンドリング | パース失敗時は該当ファイルをスキップし、エラー内容をユーザーに通知 |

## 4. 出力仕様

### 4.1 出力一覧

| # | 出力機能 | Phase | 概要 |
|---|---------|-------|------|
| 1 | N値統計・深度分布 | 1 | 地層ごとのN値統計量と深度分布グラフ |
| 2 | 支持層判定 | 1 | N値閾値に基づく支持層の自動検出 |
| 3 | 地盤定数推定 | 2 | 経験式によるφ, c, E等の推定 |
| 4 | 液状化判定用パラメータ | 2 | FL値算出に必要なパラメータ抽出 |
| 5 | 柱状図・地層断面図 | 3 | 柱状図の再描画と断面図の自動生成 |

---

### 4.2 出力1: N値統計・深度分布

#### 機能概要

ボーリングごと・地層ごとにN値の統計量を算出し、深度分布グラフとともに表示する。

#### 出力項目

| 項目 | 内容 |
|------|------|
| 統計量 | 平均値、最小値、最大値、中央値、標準偏差、データ数 |
| 集計単位 | 地層ごと、ボーリング全体 |
| グラフ | N値-深度分布散布図（Y軸: 深度（下向き正）、X軸: N値） |
| 複数ボーリング | 重ね合わせ表示対応 |

#### 画面表示

- 統計テーブル: 地層名・深度範囲・N値統計量
- 深度分布グラフ: インタラクティブ（ホバーで値表示、ズーム可能）

---

### 4.3 出力2: 支持層判定

#### 機能概要

N値の深度方向走査により、支持層と判定される区間を自動検出する。

#### 判定ロジック

| パラメータ | デフォルト値 | カスタマイズ |
|-----------|------------|------------|
| N値閾値 | 50 | ユーザー変更可能 |
| 連続回数 | 3回以上 | ユーザー変更可能 |
| 最小連続層厚 | 5m | ユーザー変更可能 |

#### プリセット

| プリセット名 | 閾値 | 連続条件 | 出典 |
|-------------|------|---------|------|
| 道路橋示方書 | N≧50 | 5m以上連続 | 道路橋示方書・同解説 IV 下部構造編 |
| 建築基礎指針 | N≧50 | 3回以上連続 | 建築基礎構造設計指針 |

#### 出力項目

| 項目 | 内容 |
|------|------|
| 支持層上端深度 | GL-○○m |
| 支持層下端深度 | GL-○○m（確認掘進含む） |
| 支持層の土質 | 土質区分・土質名 |
| 平均N値 | 支持層区間の平均N値 |
| 連続回数 | N≧閾値の連続回数 |
| 判定信頼度 | HIGH / MEDIUM / LOW |

#### 判定信頼度の基準

| レベル | 条件 |
|--------|------|
| HIGH | N≧50が5m以上連続、平均N≧60 |
| MEDIUM | N≧50が3m以上連続、一部50未満が混在 |
| LOW | 閾値付近で不安定、連続性が弱い |

---

### 4.4 出力3: 地盤定数推定（Phase 2）

#### 機能概要

N値から経験式を用いて設計用地盤定数を推定する。複数の経験式を選択可能とし、出典を明記する。

#### 推定対象パラメータ

| パラメータ | 記号 | 単位 | 適用土質 |
|-----------|------|------|---------|
| 内部摩擦角 | φ | 度 | 砂質土・礫質土 |
| 粘着力 | c | kN/m² | 粘性土 |
| 変形係数 | E | kN/m² | 全土質 |
| 単位体積重量 | γ | kN/m³ | 全土質 |
| 地盤反力係数 | kh | kN/m³ | 全土質 |

#### 主要経験式

**砂質土**

| パラメータ | 経験式 | 出典 |
|-----------|--------|------|
| φ | φ = √(12N) + 15 | Dunham (1954) |
| φ | φ = √(20N) + 15 (N≦5) / φ = 0.3N + 27 (N>5) | 道路橋示方書 |
| E | E = 2800N | 道路橋示方書 |

**粘性土**

| パラメータ | 経験式 | 出典 |
|-----------|--------|------|
| c | c = qu/2 ≒ 6.25N (kN/m²) | 一般式 |
| c | c = N/8 ～ N/4 (kgf/cm²) | 各種文献 |
| E | E = 700N | 道路橋示方書 |

#### 表示要件

- 推定値には必ず「推定」マークを付与し、使用した経験式と出典を表示する
- 適用範囲外のN値（例: N=0）に対しては推定不可と表示する
- 複数の経験式の結果を並列表示し、設計者が選択できるようにする

---

### 4.5 出力4: 液状化判定用パラメータ（Phase 2）

#### 機能概要

液状化判定（FL法）に必要な地盤パラメータをBEDデータから抽出・整理する。

#### 抽出パラメータ

| パラメータ | 記号 | データソース | 備考 |
|-----------|------|------------|------|
| 地下水位 | GWL | BED: 地下水位要素 | |
| 層厚・深度 | H, z | BED: 地層区分 | |
| N値 | N | BED: 標準貫入試験 | |
| 細粒分含有率 | Fc | BED: 粒度試験結果 | データなしの場合は土質名から概算推定 |
| 平均粒径 | D50 | BED: 粒度試験結果 | データなしの場合は土質名から概算推定 |
| 土質区分 | — | BED: 地層区分 | 砂質土のみ抽出 |
| 単位体積重量 | γ | 出力3の推定値 | |

#### 推定値の扱い

- BEDに粒度試験結果が含まれない場合、土質名からの概算値を使用する
- 概算値には「推定」ラベルを付与し、実測値との区別を明確にする
- FL値の計算は地震動条件（設計水平震度等）の入力UIを含めて対応する

### 4.7 出力6: 杭基礎設計用地盤条件整理

#### 機能概要

杭基礎設計に必要な地盤条件を、層ごとにテーブル形式で整理する。

#### 出力項目

| 項目 | 内容 |
|------|------|
| 層番号 | 上から順 |
| 深度範囲 | GL-○○m ～ GL-○○m |
| 層厚 | m |

| 土質名 | BED原文 |

## 6. データモデル

### 6.1 内部データモデル（TypeScript型定義）

```typescript
/** ボーリング1本分のデータ */
interface Borehole {
  id: string;                    // ボーリング名（B-1等）
  location: {
    latitude: number;            // 緯度（10進法）
    longitude: number;           // 経度（10進法）
    elevation: number;           // 孔口標高 (m)
    coordinate?: {               // 平面直角座標
      system: number;            // 座標系番号
      x: number;
      y: number;
    };
  };
  totalDepth: number;            // 掘進長 (m)
  waterLevel?: number;           // 地下水位 (GL-m)
  date: string;                  // 調査日
  layers: SoilLayer[];           // 地層区分
  sptResults: SPTResult[];       // 標準貫入試験結果
  metadata: {
    projectName: string;
    contractor: string;
    investigator: string;
  };
}

/** 地層区分 */
interface SoilLayer {
  index: number;                 // 層番号
  topDepth: number;              // 層上端深度 (GL-m)
  bottomDepth: number;           // 層下端深度 (GL-m)
  thickness: number;             // 層厚 (m)
  soilType: SoilType;           // 土質区分（分類後）
  soilName: string;              // 土質名（原文）
  geologicalAge?: string;        // 地質年代
  color?: string;                // 色調
  description?: string;          // 備考
}

/** 土質区分 */
type SoilType =
  | 'GRAVEL'      // 礫質土
  | 'SAND'        // 砂質土
  | 'SILT'        // シルト
  | 'CLAY'        // 粘性土
  | 'ORGANIC'     // 有機質土
  | 'ROCK'        // 岩盤
  | 'FILL'        // 盛土
  | 'UNKNOWN';

/** 標準貫入試験結果 */
interface SPTResult {
  depth: number;                 // 試験深度 (GL-m)
  nValue: number;                // N値
  penetration: number;           // 貫入量 (cm)
  blowCounts: {
    preliminary: number;         // 予備打ち
    main: [number, number];      // 本打ち
  };
  layerIndex: number;            // 対応する地層のindex
}

/** N値統計結果 */
interface NValueStatsResult {
  boreholeId: string;
  layerIndex: number;
  soilType: SoilType;
  depthRange: [number, number];
  nValues: number[];
  stats: {
    count: number;
    mean: number;
    min: number;
    max: number;
    stdDev: number;
    median: number;
  };
}

/** 支持層判定結果 */
interface BearingLayerResult {
  boreholeId: string;
  topDepth: number;
  bottomDepth: number;
  averageNValue: number;
  consecutiveCount: number;
  soilType: SoilType;
  confidence: 'HIGH' | 'MEDIUM' | 'LOW';
}

/** 杭基礎設計用地盤条件 */
interface PileFoundationResult {
  boreholeId: string;
  layers: {
    topDepth: number;
    bottomDepth: number;
    soilType: SoilType;
    averageN: number;
    designN: number;             // 設計用N値
  }[];
  bearingLayerDepth: number;
  embedmentLength: number;       // 根入れ長推奨値
}
```

### 6.2 土質区分マッピング

BEDの土質名文字列（自由記述）をSoilTypeに分類する。パターンマッチングで対応。

| パターン（正規表現） | SoilType | 例 |
|---------------------|----------|-----|
| `礫\|れき\|グラベル` | GRAVEL | 砂混じり礫、玉石混じり礫 |
| `砂\|サンド` (礫を含まない) | SAND | 細砂、中砂、細粒分まじり砂 |
| `シルト` | SILT | シルト、砂質シルト |
| `粘土\|粘性土\|クレイ` | CLAY | 粘土、シルト質粘土 |
| `有機\|腐植\|泥炭` | ORGANIC | 有機質粘土、腐植土 |
| `岩\|風化\|軟岩\|硬岩\|基盤` | ROCK | 風化岩、軟岩、花崗岩 |
| `盛土\|埋土\|表土\|造成` | FILL | 盛土、埋立土 |
| 上記に該当なし | UNKNOWN | — |

- 複合土質（例: 「砂混じり礫」）は、主要土質を優先して分類する
- UNKNOWNに分類された場合は、画面上で警告を表示しユーザーに手動分類を促す

### 8.1 パフォーマンス

| 項目 | 目標値 |
|------|--------|
| XMLパース速度 | 1ファイルあたり1秒以内 |

| 初回ページロード | LCP 2.5秒以内 |
| Excel出力 | 5秒以内 |
| PDF出力 | 10秒以内 |

### 8.2 セキュリティ

| 対策 | 内容 |
|------|------|
| XXE攻撃対策 | fast-xml-parserのエンティティ展開無効（デフォルト） |

| Rate Limiting | Vercel Edge Middlewareで実装（IP単位、100リクエスト/分） |

| HTTPS | Vercelが自動提供 |

### 9.3 リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| BED形式の調査会社間差異 | 高 | 複数社の実データでテスト。パース失敗時のフォールバック処理を充実させる |

| 柱状図描画の工数超過 | 中 | Phase 3に分離済み。MVP公開を優先する |
| Vercel無料枠の制約 | 低 | ステートレス構成により関数実行時間を短縮。超過時はPro Plan（月$20）に移行 |
