EXECUTION DECISION

今回の戦略判断

元文書は議論ログとして残す。実行版では「意思決定」「やらないこと」「担当」「期限」「完了条件」だけを見る。

90日で得るもの

福山または同等の九州準大手1社で有償プロトタイプを締結。無理なら前提を分解してリセット判断する。

ピン1

福山コンサル詳細設計部門の照査ワークフロー。代表ユーザーは渋谷さん型の個人ヘビー照査担当者。

成功定義

契約更新ではない。新規照査指摘がCiviLinkへ集約され、紙・DocuWorks並走が例外扱いになること。

成功状態

現場担当者、上司、決裁者の3者が「この照査業務はCiviLinkで回せる」と言える。効果測定データと社内稟議用レポートが顧客社内の言葉で作られている。

象徴ゴール

渋谷さんが「この照査指摘はCiviLinkにしか入れない」と部門内で宣言できる状態。これはKPIではなく、照査ワークフロー移行が起きたかを測る行動サイン。

スコープ境界（やらないこと）

「DocuWorks廃止宣言」=照査指摘の唯一の置き場宣言。数量計算書のバインド・成果品取りまとめ・束ねUXはPin1スコープ外。DocuWorks併用継続を許容し、Pin2以降に温存する。

STOP DOING

90日間やらないこと

例外は、ピン1の照査ワークフロー移行に直接効くものだけ。判断基準は「照査指摘の唯一の置き場宣言（=「DocuWorks廃止宣言」）の阻害要因を消すか」。数量計算書バインド・成果品取りまとめ等の束ねUXは対象外。

新規の無料トライアル提案をしない
全国デモ行脚をしない
「興味あり」案件をパイプラインに置かない
若手技術者ペルソナを並列で走らせない
OCR/LLM/自治体/施工会社を同じ会議で混ぜない
福山専用要望を根拠なく全体ロードマップ化しない
